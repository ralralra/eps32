# AI Studio 프롬프트 원문 예시 — 매점팀 버전

> 다른 팀은 **화면·action 이름만** 자기 것으로 교체 — 구조는 그대로!
> 프롬프트도 팀의 산출물 — AI Studio에 붙여넣기 전에 문서로 먼저 저장하세요.

```
학교 매점 관리 웹앱(PWA)을 만들어줘.

화면 1 · 판매 화면 :
- 제목 "[3팀] 스마트 매점"
- 상품 카드 6개 : 이름, 가격, 남은 재고, [판매] 버튼
- [판매]를 누르면
  [웹앱 URL]?action=sell&item=상품명 으로 요청하고
  "판매 완료" 메시지를 잠깐 보여줘

화면 2 · 통계 화면 :
- 오늘 판매 수와 인기 상품 순위 표시
- [웹앱 URL]?action=stats 에서 가져와 표시
- 화면 1과 탭으로 전환되게 해줘

공통 디자인 : 파란 계열, 모바일 우선, 버튼 크게, 글자 크게
```

## 포인트

- 화면별로 나눠 쓰면 AI가 **화면 전환까지** 만들어 줘요
- `action`·`item`의 철자는 시트의 상품명·Apps Script 분기와 **완전히 같아야** 해요 (4회차 '이름 대조표'!)
- 재고 경고·소리 등 **하드웨어 반응은 프롬프트에 넣지 않아요** — 그건 시트 명령 셀과 ESP32의 몫
- 완성되면 4회차 방법으로 팀원 전원 폰에 PWA 설치

## 팀별 교체 가이드

| 팀 | 화면 1 | 화면 2 | action 예시 |
|---|---|---|---|
| ① 매점 | 상품 카드 + [판매] | 판매 수·인기 순위 | `sell`, `stats` |
| ② 키오스크 | 메뉴 + 장바구니 + [주문] | 주문번호·매출 통계 | `order`, `sales` |
| ③ 우산 | QR 스캔 + [대여]/[반납] | 대여 현황·미반납 | `rent`, `return`, `status` |
| ④ 화분 | 현재 수분·온습도 카드 | 기록 그래프·급수로그 | `latest`, `history`, `water` |
| ⑤ 출석 | QR 체크인 + 현재 인원 | 출석부·통계 | `checkin`, `checkout`, `list` |

> 💡 QR 스캔이 필요한 팀(우산·출석): "카메라로 QR을 스캔하는 기능을 추가해줘"라고 지시 (html5-qrcode 방식) — 6회차에 연결해요.
> 💡 심화: 키 없는 공개 API(Open-Meteo 등)를 쓰고 싶은 팀은 "~API를 사용해서 …도 표시해줘" 한 문장 추가 (4회차 심화 참고).
