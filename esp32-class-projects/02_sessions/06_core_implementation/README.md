> 웹 터치 한 번이 하드웨어를 움직이고 데이터로 남는, 끊김 없는 한 줄기 흐름을 만들어요.

- [ ] 기능 동결을 선언하고, 여유가 있으면 AI 선택 기능에 도전한다

## 1교시 — 배선 확정과 캘리브레이션

센서를 믿으려면, 먼저 센서를 의심해야 해요.

### 오늘의 개발 전략 — 처음부터 다시 쓰지 않아요

1. **동작하는 상태를 유지하며 조금씩** — 한 번에 왕창 고치면 어디서 틀렸는지 찾을 수 없어요. 고치고, 확인하고, 또 고쳐요
2. **4회차 템플릿에서 바꿀 곳은 3군데뿐**:
   ① ESP32의 센서 읽기 부분 ② Apps Script의 action 함수들 ③ 웹앱 화면
3. 실제 개발자들도 똑같이 일해요 — 잘 되는 코드에서 출발하는 것은 부끄러운 일이 아니라 **'전문가의 방식'**

### 캘리브레이션 — 기준값 측정 (전 팀 공통 원리!)

센서 원시값은 그대로 못 써요 — **기준을 잡아 해석하는 것, 이게 데이터 다루기의 본질.**

화분팀 예시 → [`code/01_soil_calibration/01_soil_calibration.ino`](code/01_soil_calibration/01_soil_calibration.ino)

1. 토양습도 셀을 실드의 **A2~A5 위치 포트**에 (WiFi 사용 중 다른 아날로그 자리(ADC2)는 먹통!)
2. 1단계 — 원시값 관찰: 공기 중(건조) 값 기록 → 물컵 속(습윤) 값 기록
3. 2단계 — 0~100%로 번역: `map(raw, 건조값, 습윤값, 0, 100)` + `constrain`
4. 실제 화분 흙에 꽂고 현재 % 기록 — 우리 화분의 '오늘 상태'가 첫 데이터!

| 팀 | 측정할 기준값 |
|---|---|
| ① 매점 | 무게 영점 + 상품 1개 무게 |
| ② 키오스크 | (센서 캘리브레이션 대신) 주문번호 발급 규칙 확정 |
| ③ 우산 | 초음파·무게 기준값 (우산 있음/없음) |
| ④ 화분 | 토양습도 건조값·습윤값 |
| ⑤ 출석 | 홀센서·자석 감지 거리 |

> ✓ 값이 반대로 움직이면(젖을수록 커짐) map의 건조값·습윤값 **순서를 서로 바꾸면** 돼요.
> 📸 배선 사진 + 건조/습윤 시리얼 캡처를 남겨요 — **기록이 발표 때 신뢰의 근거!**

## 2교시 — 핵심 시나리오: 엔드 투 엔드

### 엔드 투 엔드(End to End)란

끝(사용자 터치)에서 끝(데이터 저장)까지 **전 구간이 연결된 상태.**

1. **반쪽 완성의 함정**: 웹만 예쁘거나, 보드만 움직이면 아직 0점 — 연결되어야 시스템!
2. **오늘의 합격선**: 핵심 시나리오 '단 1개'가 전 구간 관통 — **여러 기능 어중간보다 한 기능 완벽이 나아요**
3. **검증 방법**: 폰에서 조작 → 3초 안에 장치 반응 → 시트에 새 줄 확인. 세 가지가 모두 보이면 통과

### 팀별 핵심 시나리오 — 오늘 뚫어야 할 한 줄기

| 팀 | 핵심 시나리오 | 하드웨어 반응 | 시트 기록 |
|---|---|---|---|
| ① 매점 | 웹에서 [콜라 판매] 터치 → 재고 1 차감 | LCD 'SOLD: cola' + LED 깜빡 | 판매로그 1줄 |
| ② 키오스크 | 장바구니 [주문] → 주문번호 발급 | LCD에 주문번호 크게 표시 | 주문로그 1줄 |
| ③ 우산 | 웹 QR 스캔 → 대여 처리 | 무게 변화로 대여 확인 + LCD 안내 | 대여로그 1줄 |
| ④ 화분 | 수분 20% 미만 감지 → 급수 알림 | LCD '급수 필요!' + 웹 경고 | 알림·급수로그 1줄 |
| ⑤ 출석 | 웹 체크인 → 인원 +1 | LCD 인원 갱신 + LED 깜빡 | 출입로그 1줄 |

> 💡 QR 스캔은 웹에서: AI Studio에 "카메라로 QR을 스캔하는 기능을 추가해줘"라고 지시 (html5-qrcode 방식)

### 연결 요령 — action 하나를 끝까지 뚫기

매점팀 예시 (다른 팀도 이름만 바꾸면 같은 구조) → [`code/02_sell_end_to_end.md`](code/02_sell_end_to_end.md)

- 웹은 action만 던져요 — **판단은 서버(Apps Script)가**
- 하드웨어 알림은 **명령 셀(A1)을 통해** 전달 (4회차 폴링 그대로)
- **처리 후 A1을 비워야 알림이 한 번만 울려요**
- 한 구간씩 시리얼 모니터·시트를 보며 확인

### ✓ 엔드 투 엔드 3종 증거 (합격 판정법)

한 번의 터치가 **세 화면에 동시에** 나타나야 해요. 역할 분담: 조작 1명 · 촬영 1명 · 시트 감시 1명.

- [ ] 증거 ① 폰 조작 화면 캡처
- [ ] 증거 ② 장치 반응 사진/영상 (발표 백업 영상으로도 사용!)
- [ ] 증거 ③ 시트에 추가된 새 줄 (시각까지 함께 보이게)

> 세 증거의 시각이 같은 1분 안에 있으면 **엔드 투 엔드 공식 인증** 🏅

## 3교시 — 데이터가 말하게 하라: 통계와 알림

**기록의 목적은 저장이 아니라 '결정'이에요.**

### Apps Script 집계 — 인기 순위 만들기

판매로그를 세면 인기 순위가 돼요 → [`code/03_stats_function.gs`](code/03_stats_function.gs)

- `getValues()`: 시트 전체를 표(2차원 배열)로 읽기
- for문으로 한 줄씩 세기 — **엑셀 COUNTIF의 코드 버전**
- 결과를 JSON으로 응답 → 웹앱이 받아 순위 표시
- 응용: 화분팀=최근 24시간 평균, 우산팀=이용 시간 계산
- **CSV는 공짜**: 시트에서 `파일` → `다운로드` → `CSV` — 발표 때 '데이터 파일'로 제출!

### 알림 설계 — 빛과 글자에도 '문법'이 있어요

상태별 알림 표를 정하면 사용자가 배우지 않아도 이해해요:

| 상황 | LED 패턴 | LCD 문구 | 웹 표시 |
|---|---|---|---|
| 성공 (판매·출석·반납 완료) | 짧게 2회 | `OK` 문구 | 완료 메시지 |
| 경고 (재고 부족·수분 부족·우산 없음) | 빠른 연속 점멸 | `!!` | 빨간 경고 |
| 안내 | — | 1줄=상태 요약, 2줄=상세 (`SOLD: cola / stock 6`) | — |

- **소리 알림은 폰이 담당**: 웹앱 프롬프트에 '알림 소리를 재생해줘' 한 줄 — 부저 없이도 폰이 스피커!
- 이 표를 팀별로 정해 발표 자료에 포함 — '왜 이렇게 정했나'를 설명할 수 있으면 최고

## 4교시 — 중간 데모 · 기능 동결 · AI 심화

**멈출 줄 아는 것도 실력이에요.**

### 중간 데모 (팀당 3분)와 기능 동결

1. **데모 형식**: 핵심 시나리오를 '폰 → 장치 → 시트' 순서로 실연 — 설명은 짧게, 동작은 확실하게
2. **피드백 규칙**: 다른 팀은 '좋은 점 1 + 개선 제안 1'을 포스트잇으로 — 비난 금지, 구체적으로
3. **기능 동결(Feature Freeze) 선언**: 지금부터 새 기능 추가 금지! 남은 시간은 오류 수정과 다듬기만
4. 왜 동결? **발표 전날 새 기능을 넣다가 전체가 깨지는 사고** — 개발 역사상 가장 흔한 비극이에요

### 여유 팀 도전 — AI 스테이션 (선택·심화)

블록코딩 도구의 AI 기능: PC+웹캠으로 '인식하는 부스'를 병렬 운영.

| 팀 | AI 선택 기능 | 방식 | 주의 |
|---|---|---|---|
| ① 매점 | 과자를 카메라에 대면 상품 자동 인식 | 이미지 분류 (직접 촬영·학습) | 각 상품 사진 20장+ |
| ② 키오스크 | "콜라 주세요" 음성 주문 | 음성 인식 블록 | 조용한 곳에서 시연 |
| ④ 화분 | 잎 사진으로 건강/시듦 판정 | 이미지 분류 | 급수 판단의 보조로만 |
| ⑤ 출석 | 이미지 인식 체크인 체험 | 이미지 분류 | **사진 저장 금지·사전 동의** |

> 핵심 학습: 팀이 직접 데이터를 모아 모델을 학습시키는 경험 — 'AI는 데이터로 만들어진다'를 몸으로. 단, **핵심 시스템 완성이 언제나 우선!**

## 🔧 자주 막히는 문제 해결소 — 6회차

| 증상 | 해결 |
|---|---|
| 웹 버튼 → 장치 반응이 안 와요 | **구간 분리 점검**: ① `URL?action=…` 직접 열기 ② 시트 명령 셀 변하나 ③ ESP32 시리얼에 폴링 로그 찍히나 — 끊긴 구간만 고쳐요 |
| 알림이 계속 반복해서 울려요 | 처리 후 명령 셀(A1)을 비우는 코드가 빠짐 — `setValue("")` 추가 |
| 토양수분 % 값이 반대로 나와요 | 저항식 센서는 건조할수록 값이 큰 경우가 많아요 — map의 건조값·습윤값 순서를 바꾸면 해결 |
| 통계 숫자가 이상해요 | `getValues()`의 0번째 줄은 '제목 행' — for문을 `i=1`부터 시작했는지 확인 |
