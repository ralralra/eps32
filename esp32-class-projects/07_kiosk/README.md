# 스마트 창업 키오스크

과자를 파는 **무인 키오스크**입니다. 폰/태블릿 화면에서 상품을 골라 담고,
**가상 포인트로 모의결제**한 뒤 매출이 자동 집계됩니다. (창업 교육 연계)

## 무엇을 만드나

```
폰/태블릿 화면(키오스크)  ──▶  ESP32  ──▶  포인트 차감(모의결제) / 판매 기록
 상품 선택 → 장바구니 → 결제        │
                                    └──▶ 매출 집계 대시보드
```

- **추가 부품 없음** — Wemos D1 R32만. 화면은 폰/태블릿 브라우저
- **실제 돈 안 씀** — 가상 포인트로 "결제 흐름"만 체험 (기획서 원칙)

> 🧭 **보드 안내**: 이 프로젝트는 **Wemos D1 R32** 기준입니다. 추가 부품이 없어 보드만 바꾸면 그대로 동작해요.
> - 처음 세팅: [`01_docs/setup_wemos_d1_r32.md`](../01_docs/setup_wemos_d1_r32.md)
> - 고릴라셀 확장 아이디어: **7세그 셀로 주문번호 표시 + 소리센서 활용** → [6회차 — 키오스크팀 핵심 시나리오](../02_sessions/06_core_implementation/README.md)

## 단계 안내

| 단계 | 폴더 | 무엇을 / 확인 |
|---|---|---|
| 1 | `step1_menu_browser` | 키오스크 화면(상품·장바구니·결제) — **PC 브라우저** |
| 2 | `step2_esp32_menu` | 같은 화면을 ESP32가 보여주기 — 폰/태블릿 |
| 3 | `step3_pay_points` | 가상 포인트 모의결제 + 판매 기록 — 폰 |
| 4 | `step4_sales_dashboard` | 매출·인기상품 집계 보기 — 폰 |

> 출석 프로젝트와 똑같은 "브라우저에서 먼저 → ESP32로 이식" 흐름이에요.

## 준비물
- Wemos D1 R32 + USB
- 폰 또는 태블릿 (키오스크 화면용 — 태블릿이 더 키오스크답습니다)

## 매점 프로젝트와의 관계
- 매점(`05_store/`) = RFID로 **계산원 없는 셀프 계산**
- 키오스크(`07_kiosk/`) = 화면으로 **상품 선택 + 모의결제**
- 데이터 구조(상품·판매로그)가 비슷해서 한 팀이 묶어서 하기 좋아요
