# 1단계 — RFID 카드 UID 읽기

## 목표
카드를 리더에 대면 그 카드의 **고유번호(UID)** 가 시리얼에 뜨게 한다. 각 상품 카드의 UID를 적어둔다.

## 원리
모든 RFID 카드에는 공장에서 새겨진 **UID(고유번호)** 가 있어요. 지문 같은 거예요.
이 UID를 "어떤 상품인지" 구분하는 열쇠로 씁니다.

```cpp
if (rfid.PICC_IsNewCardPresent() && rfid.PICC_ReadCardSerial()) {
  // rfid.uid.uidByte[] 에 고유번호가 들어있음
}
```

## 준비
- RC522 배선 ([`../docs/wiring_and_setup.md`](../docs/wiring_and_setup.md) — SDA=5, RST=27, **3.3V**!)
- MFRC522 라이브러리 설치

## 해보기
1. `step1_read_card.ino` 업로드
2. 시리얼 모니터(115200)
3. 카드를 한 장씩 리더에 대기
4. 뜨는 UID를 **카드별로 적어두기** (예: 초코파이 = `A1B2C3D4`)
5. "결제용" 카드 1장도 UID 적어두기

## 확인 ✅
- [ ] 카드를 대면 `카드 UID: ########` 가 뜬다
- [ ] 카드마다 UID가 다르다
- [ ] 같은 카드는 항상 같은 UID가 나온다
- [ ] 상품 카드 + 결제 카드의 UID를 모두 적었다

## 잘 안 될 때
- 반응 없음 → 배선(특히 3.3V!), 라이브러리 설치 확인
- 학생증이 안 읽힘 → MIFARE Classic이 아닐 수 있음 → 별도 카드 사용

## 배운 것
- RFID = 카드의 고유번호(UID)를 무선으로 읽는 기술
- SPI 통신으로 리더(RC522)와 ESP32가 대화한다
- UID를 키로 쓰면 "이 카드 = 이 상품/사람" 으로 연결 가능
