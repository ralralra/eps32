# 5단계 — 출석 버튼 데이터를 ESP32로 보내기

## 목표
폰에서 이름 넣고 출석 → 그 이름이 **ESP32로 전달** 되어 시리얼에 `출석: 홍길동` 이 찍힌다.

## 핵심 아이디어
4단계는 ESP32 → 폰 (페이지 보내기, 한 방향)이었어요.
이번엔 반대로 **폰 → ESP32 (데이터 보내기)** 를 추가합니다.

이 "보내기"를 하는 명령이 **fetch** 예요.
```
폰: "야 ESP32, 이름이 홍길동이래"   →   fetch("/check?name=홍길동")
ESP32: "응 받았어, 출석 완료!"      →   답장
```

## 코드에서 새로 생긴 것

폰 쪽 (웹페이지):
```javascript
fetch("/check?name=" + encodeURIComponent(name))   // ESP32로 이름 보내기
  .then(r => r.text())
  .then(answer => { msg.innerText = answer; });     // ESP32의 답을 화면에
```

ESP32 쪽:
```cpp
void handleCheck() {
  String name = server.arg("name");   // 보내온 이름 꺼내기
  Serial.println("출석: " + name);    // 시리얼에 출력
  server.send(200, "text/plain; charset=utf-8", name + "님 출석 완료!");
}
...
server.on("/check", handleCheck);     // /check 주소를 처리하도록 등록
```

> `encodeURIComponent` = 한글/공백을 인터넷으로 보낼 수 있는 형태로 바꿔주는 안전장치

## 해보기
1. 업로드 후 시리얼 모니터(115200) 열어두기
2. 폰으로 `ESP32-Attend` 접속 → 192.168.4.1
3. 이름 넣고 "출석하기"
4. **폰 화면**에 "홍길동님 출석 완료!" (초록 글씨) 뜨는지 확인
5. **시리얼 모니터**에 `출석: 홍길동` 찍히는지 확인

## 확인 ✅
- [ ] 출석 누르면 폰에 "○○님 출석 완료!" 가 뜬다
- [ ] 동시에 시리얼 모니터에 `출석: ○○` 이 찍힌다
- [ ] 여러 명 해보면 시리얼에 계속 쌓인다

## ⚠ 아직 문제
전원을 껐다 켜면? 시리얼에 찍힌 기록은 사라집니다.
ESP32가 이름을 **받기만 하고 저장은 안 했기** 때문이에요.
→ 다음 단계에서 보드 안에 저장합니다. (3단계의 "저장 개념"이 여기서 다시!)

## 배운 것
- fetch = 웹페이지가 서버(ESP32)에 데이터를 보내는 방법
- `server.arg("name")` = 보내온 값 꺼내기
- `server.on("주소", 함수)` = 주소마다 다른 처리 연결
- 받기는 했지만 저장은 아직 (다음 단계)
