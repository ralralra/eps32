# 4단계 — 매점 웹 대시보드 (완성본) 🎉

## 목표
폰으로 접속하면 **재고 현황 · 총 매출 · 상품별 판매수**를 한눈에 본다.
RFID 셀프 계산(1~3단계) + 웹 조회(출석에서 배운 웹서버)를 합친 완성본.

## 핵심 — 두 기술의 합체
이 단계는 지금까지 배운 걸 **합치는** 단계예요:
- RFID 읽기 (매점 1~3단계)
- 웹서버로 보여주기 (출석 4~7단계)
- 파일 저장/집계 (출석 6단계 + 매점 3단계)

```cpp
void loop() {
  server.handleClient();          // 웹 요청 처리 (출석에서 배움)
  // + RFID 카드 읽어서 판매 처리 (매점에서 배움)
}
```

## 새로 보는 것 — 집계
판매기록 파일을 읽어서 **상품별 판매 수와 총 매출을 계산**:
```cpp
void readStats(int soldCount[], int &grandTotal) {
  // /sales.csv 를 한 줄씩 읽어 상품별로 세고 가격을 더함
}
```

## 해보기
1. 상품표 UID를 내 카드로 수정
2. Partition: Default 4MB with spiffs, 업로드
3. 폰으로 `ESP32-Store` 접속 (비번 `store1234`) → http://192.168.4.1
4. 다른 손으로 상품 카드를 찍어 판매 발생시키기
5. 폰 화면(3초마다 새로고침)에서 재고 ↓, 판매수 ↑, 매출 ↑ 확인

## 확인 ✅
- [ ] 폰에 재고/매출/판매수 표가 보인다
- [ ] 카드로 팔면 화면 숫자가 갱신된다
- [ ] 재고 2개 이하면 ⚠ 표시
- [ ] 전원 껐다 켜도 매출/판매기록이 남아있다

## 잘 안 될 때
- 웹 접속 안 됨 → 출석 트러블슈팅과 동일 (http, IO0+RESET 등)
- 카드 안 읽힘 → 매점 [`docs/wiring_and_setup.md`](../docs/wiring_and_setup.md) (3.3V, 배선)
- WiFi 켜니 카드가 안 읽힘 → 거의 없음. 그래도면 RC522 배선/전원 확인

## 배운 것 (매점 전체 정리)
1. RFID UID 읽기
2. UID ↔ 상품 표로 연결, 가격 합산
3. 재고 관리 + 판매 기록 저장
4. 웹으로 매출·재고·인기상품 집계 조회

## 확장 아이디어
- 인기상품 1위 강조, 막대그래프
- 유통기한 등록 → D-day 폐기 알림 (기획서 항목)
- 포인트카드로 모의결제 → **키오스크 프로젝트로 연결**
- 판매 시각 기록 (폰 시각 활용, 출석 6단계 방식)
