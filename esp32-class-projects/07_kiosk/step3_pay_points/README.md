# 3단계 — 가상 포인트 모의결제 + 판매 기록 (ESP32에서)

## 목표
포인트 잔액과 판매 기록을 **ESP32(기기)** 에 둔다. 어느 폰으로 접속하든 **같은 잔액·기록을 공유**하고, 전원 꺼도 유지된다.

## 핵심 아이디어 — 결제를 기기로 옮기기
2단계는 포인트가 각 폰 안에 따로 있었죠. 이제 ESP32 한곳에 둡니다.
```
폰: "이 장바구니 결제해줘"  →  fetch("/pay?cart=새우깡:700|콜라:1000")
ESP32: 잔액 확인 → 차감 → 판매 기록 → "OK, 남은잔액" 회신
```
이게 진짜 결제 시스템의 구조예요 — **잔액과 기록은 서버(기기)가 책임진다.**

## 새로 보는 것
- `/balance` : 현재 잔액을 알려주는 주소
- `/pay?cart=...` : 결제 처리 (잔액 차감 + 판매 기록)
- 장바구니를 `새우깡:700|콜라:1000` 문자열로 보내 ESP32가 분리해서 처리

## 해보기
1. Partition: Default 4MB with spiffs, 업로드
2. 폰으로 `ESP32-Kiosk` 접속 → http://192.168.4.1
3. 상품 담고 결제 → "결제 완료! 잔액 ...P"
4. **다른 폰**으로도 접속해보기 → 같은 잔액이 보임 (공유됨!)
5. 전원 껐다 켜도 잔액이 유지되는지 확인
6. 포인트보다 많이 담아 결제 → "포인트가 부족합니다!"

## 확인 ✅
- [ ] 결제하면 기기의 잔액이 줄어든다
- [ ] 다른 폰에서도 같은 잔액이 보인다 (공유)
- [ ] 전원 재시작해도 잔액·기록이 남는다
- [ ] 잔액 부족 시 결제가 막힌다

## 배운 것
- 잔액·기록을 서버(기기)가 책임지면 여러 사용자가 공유 가능
- 장바구니 데이터를 문자열로 주고받고, 서버가 분리해서 처리
- 출석/매점의 저장 개념이 결제에도 그대로
