# 4단계 — 매출 집계 대시보드 (완성본) 🎉

## 목표
쌓인 판매 기록으로 **총매출 · 상품별 판매수**를 집계해서 사장님 화면으로 본다.

## 두 화면
```
손님:   http://192.168.4.1/        키오스크 (상품선택·결제)
사장님: http://192.168.4.1/sales   매출 집계 + 초기화
```

## 핵심 — 집계
판매기록(`/sales.csv`)을 한 줄씩 읽어 상품별로 세고 가격을 더해요. (매점 4단계와 같은 방식)
```cpp
for (각 판매줄) { revenue += price; 그 상품 count++; }
```

## 해보기
1. Partition: Default 4MB with spiffs, 업로드
2. 폰으로 키오스크 접속 → 여러 번 결제해서 판매 발생
3. 화면 맨 아래 "(사장님) 매출 보기" → `/sales`
4. 총매출과 상품별 판매수가 맞는지 확인
5. "매출 초기화"로 판매기록 리셋 (포인트 잔액은 유지)

## 확인 ✅
- [ ] 매출 페이지에 총매출·상품별 판매수가 보인다
- [ ] 결제할수록 숫자가 늘어난다
- [ ] 전원 껐다 켜도 매출 기록이 남는다
- [ ] 초기화가 동작한다

## 배운 것 (키오스크 전체 정리)
1. 키오스크 화면 만들기 (브라우저)
2. ESP32가 화면 제공
3. 가상 포인트 모의결제 + 판매 기록 (기기에서)
4. 매출 집계 조회

## 확장 아이디어
- 인기상품 1위 강조, 막대그래프
- 상품별 재고 추가 (매점 방식) → 품절 처리
- RFID 포인트카드로 결제 (매점 RFID + 키오스크 결제 합치기)
- 포인트 충전 화면 (사장님용)
- 서보모터로 상품 배출 데모 (기획서 항목)
