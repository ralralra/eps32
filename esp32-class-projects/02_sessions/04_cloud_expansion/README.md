## 1교시 — 구글이 우리의 서버가 돼요

### 오늘 완성할 새 구조 (구조 B)

폰과 ESP32는 직접 만나지 않아요 — 구글 시트라는 '**공용 게시판**'으로 소통해요.

```
📱 스마트폰 (어디서든)   ← 인터넷으로 접속
      ▼
🌐 AI Studio 웹앱        ← 화면 담당 (3교시 제작)
      ▼
⚙️ Apps Script           ← 서버 담당 — '구글 위의 server.on()'
      ▼
📊 Google Sheets         ← 저장 담당 — 데이터베이스!
      ▼
📡 ESP32 + 센서          ← 10초마다 업로드 & 명령 확인 (폴링)
```

무엇이 좋아질까?
- 시트에 쓰면 **영원히 남아요** → 통계·CSV 해결
- 구글 서버는 24시간 켜져 있어요 → **집에서도 접속**
- 지난주 배운 요청-응답 원리가 그대로 반복돼요

### 따라하기 ① 구글 시트 + Apps Script 열기

1. 구글 드라이브 → 새로 만들기 → **Google 스프레드시트**
   - 이름: `팀1_데이터` / 1행에 제목: `시각 | 온도 | 습도 | 조도 | 소음`
   - ⚠ 탭(시트) 이름이 `시트1`인지 확인 — 코드의 `getSheetByName`과 같아야!
2. 시트 메뉴: `확장 프로그램` → `Apps Script` — 여기가 우리의 '서버 공장'
3. 기본으로 있는 `function myFunction(){}` 전체 삭제
4. 프로젝트 이름을 `팀1_서버`로 변경, 저장은 Ctrl+S 수시로!

> Apps Script = 구글 문서 안에서 돌아가는 JavaScript. 구글이 컴퓨터를 빌려주는 셈 — 무료로, 24시간.

### 따라하기 ② 첫 서버 코드 — doGet

3회차의 `server.on("/")`과 완전히 같은 역할 — 자리만 ESP32에서 구글로 옮겼어요.
→ [`code/01_apps_script_basic.gs`](code/01_apps_script_basic.gs)

- `doGet` = '요청이 오면 실행되는 함수'
- `주소?temp=25&light=800…` → `e.parameter`로 꺼내요
- `appendRow` = 시트에 한 줄 쓰기
- 마지막 `return`이 응답("OK")

### 따라하기 ③ 웹앱 배포 — 오늘의 최대 고비! ⚠

설정 하나만 틀려도 전부 안 돼요 — 순서대로:

1. 우측 상단 `[배포]` → `[새 배포]` 클릭
2. 톱니바퀴 → 유형: **'웹 앱'** 선택
3. 액세스 권한: ★ **'모든 사용자'** ★ 로 변경
   - 기본값 '나만'이면 ESP32가 접속을 거부당해요 — **오늘 실패 원인 1위!**
4. `[배포]` → 권한 승인 (경고창: 고급 → 이동 → 허용) — 내가 만든 앱이라 안전, 구글의 형식적 확인 절차
5. **'웹 앱 URL' 복사 → 팀 메모장에 보관** — `https://script.google.com/...` 이 긴 주소가 우리 서버의 주소!
   - ✓ URL 끝이 **`/exec`** 인지 확인 — `/dev`로 끝나는 주소는 ESP32에서 동작하지 않아요

### 서버 개통 테스트 (ESP32 연결 전에, 브라우저로 먼저! — 구간 점검)

1. 메모장에 주소 조립: `복사한URL?temp=25&humi=48&light=800&sound=120`
2. 브라우저 주소창에 붙여넣기 → 화면에 `OK` 확인
3. 시트 탭으로 전환 → **시각·25·48·800·120이 자동으로 기록된 새 줄** 확인 🎉
4. 숫자를 바꿔 한 번 더 → 줄이 또 늘어나는지
5. 여기까지 되면 서버는 완성 — 남은 건 ESP32뿐!

> 🔧 '권한이 없습니다'가 뜨면 → 액세스 권한 재설정 후 `[배포 관리]`에서 **새 버전으로 재배포**

## 2교시 — ESP32, 클라우드에 접속하다

이번엔 폰이 아니라 **보드가 '손님'**이 되어 구글에 요청을 보내요.

### 따라하기 ④ Station 모드 + 클라우드 업로드

AP 모드(WiFi 만들기) → **Station 모드(WiFi 접속하기)**로 전환
→ [`code/02_station_upload/02_station_upload.ino`](code/02_station_upload/02_station_upload.ino)

- `WiFi.begin` = 교사 핫스팟에 접속 (이름·비번은 칠판 공지, ⚠ 2.4GHz만!)
- 주소 뒤에 `?temp=…&humi=…` 를 붙여 데이터 전달
- **`setFollowRedirects`**: 구글 특유의 '재안내'를 따라가는 옵션 — **빼먹으면 실패!**
- 10초마다 반복 → 시트에 데이터가 계속 쌓여요
- 업로드 후 시트를 열어두고 지켜봐요 — 10초마다 한 줄씩 '자동으로' 생기는 순간, 박수! 👏

### 따라하기 ⑤ 키트의 다른 센서도 함께 올리기

**센서 추가 공식 = 읽기 한 줄 + 주소 한 조각** — 어떤 센서든 똑같아요:

1. `analogRead` 한 줄 (`int l = analogRead(LIGHT);`)
2. 주소에 `&이름=` 한 조각 (`+ "&light=" + String(l)`)
3. 시트 1행에 열 하나
4. doGet에 parameter 한 줄

> 🔑 토양습도 셀(화분팀)은 **A2~A5 위치 포트(GPIO 34·35·36·39)** — WiFi 사용 중 다른 아날로그 자리는 먹통!

### ⚠ 다르면 안 되는 이름들 — 철자 하나로 전체가 멈춰요 (오류의 80%!)

| 연결 고리 | 한쪽 | 다른 쪽 | 다르면 생기는 일 |
|---|---|---|---|
| 시트 탭 이름 | 시트 하단 탭 '시트1' | `getSheetByName("시트1")` | '시트 없음' 에러 |
| 데이터 이름 | ESP32 주소의 `temp·humi·light·sound` | doGet의 `e.parameter.temp` … | 시트에 빈 칸만 기록 |
| 시트 열 순서 | 1행: 시각·온도·습도·조도·소음 | `appendRow([시각, t, h, l, s])` | 값이 엉뚱한 열에 |
| 서버 주소 | 배포된 웹앱 URL (끝이 `/exec`) | ESP32 코드 · 웹앱 fetch의 URL | 아무 반응 없음 |
| 명령 값 | 시트 A1의 `ON` / `OFF` | ESP32의 `if (cmd == "ON")` | LED 무반응 (대소문자!) |
| 요청 종류 | 웹앱이 보내는 `mode=cmd·latest` | doGet의 mode 분기 문자열 | unknown 응답 |
| 화면 이름표 | HTML의 `id="t"` | `getElementById("t")` | 화면 숫자가 안 바뀜 |

> 재배포하면 URL이 바뀔 수 있어요 — URL을 쓰는 **모든 곳(ESP32 코드·웹앱 fetch)을 한 번에 같이** 교체!

## 3교시 — 원격 제어 + AI Studio

### 따라하기 ⑥ 명령 폴링 — 반대 방향

시트 '설정' 탭 A1칸에 `ON`/`OFF`를 쓰면 → ESP32가 읽어서 실행
→ [`code/03_apps_script_full.gs`](code/03_apps_script_full.gs) + [`code/04_esp32_cloud_template/04_esp32_cloud_template.ino`](code/04_esp32_cloud_template/04_esp32_cloud_template.ino)

- 시트 A1에 `ON` 입력 → **최대 10초 뒤** 교실 LED 점등!
- 즉시가 아닌 이유 = 폴링 주기(10초)만큼 기다림
- 빠른 반응이 필요하면 주기를 3~5초로 — 대신 요청 횟수 증가 (트레이드오프)

### 따라하기 ⑦ AI Studio — 화면을 '생성'해요

2회차에 손으로 쓰던 HTML·CSS·JS를 AI가 대신 써 줘요 — **우리는 읽고 고칠 줄 알아요!**
📖 전체 가이드(New App → 프롬프트 샘플 → 폰 설치): [`01_docs/ai_studio_webapp_guide.md`](../../01_docs/ai_studio_webapp_guide.md)

1. `aistudio.google.com` 접속 → **Build** → **New App**(새 앱 만들기) 선택 (팀 구글 계정)
2. 프롬프트 입력 — 구체적으로! → [`code/05_ai_studio_prompt.md`](code/05_ai_studio_prompt.md)
3. 생성된 화면 확인 → 마음에 안 들면 추가 지시 ("온도 카드를 더 크게" — **한 번에 하나씩!**)
4. 코드 보기로 HTML·CSS·JS 구조 확인 — 2회차에 배운 3형제가 그대로! AI 결과를 '검토'할 수 있는 사람이 진짜 개발자
5. fetch 주소를 우리 웹앱 URL로 연결

> AI 활용의 핵심 = **요구사항을 명확한 문장으로 쓰는 능력.** 모호하게 시키면 모호한 결과가 나와요.

### 따라하기 ⑧ 웹앱을 폰에 '앱처럼' 설치 (PWA)

1. AI Studio에서 공유/배포 버튼 → 앱 링크 만들기
2. 링크를 팀 채널로 전송 (또는 QR 생성기로 QR 만들어 화면에 띄우기)
3. 폰 브라우저로 링크 열기 → 동작 확인
4. **홈 화면에 추가** — 안드로이드: 메뉴(⋮) → '홈 화면에 추가' / 아이폰: 공유(⬆) → '홈 화면에 추가'
5. 홈 화면 아이콘 터치로 실행 — 주소창 없는 전체 화면, 겉보기엔 완전한 앱! 📱

> ✓ 웹앱을 수정해도 링크가 같으면 재설치 불필요 — 폰에서 새로고침만 하면 최신 화면.

### 심화 — 공개 API로 바깥 날씨 붙이기

API = 서비스가 열어 둔 '데이터 창구'. 주소로 요청하면 JSON으로 응답 — **우리 doGet과 같은 원리!**

- **Open-Meteo API**: 회원가입·키가 필요 없는 무료 날씨 API — 학교 실습에 최적
- 프롬프트 예: "Open-Meteo API를 사용해서 위도 [37.57] 경도 [126.98]의 현재 기온·습도를 '바깥 날씨' 카드로 추가해줘. 교실 온도 카드와 나란히 배치해서 안팎이 비교되게. 10분마다 갱신."
- '교실 안(우리 센서) vs 바깥(API)' 비교는 발표에서 반짝이는 포인트!

> 우리도 오늘 API를 만든 거예요 — 우리 웹앱 URL이 곧 '우리 팀의 API 주소'.

## 4교시 — 미니 프로젝트: 교실 환경 모니터 완성

### 🎯 미션: 교실 환경 모니터

**목표**: 폰 웹앱에서 교실 온습도 확인 + LED 원격 제어 + 모든 기록이 시트에 저장 + CSV 다운로드까지

완성 조건:
- [ ] ESP32 → 시트 자동 업로드 (10초)
- [ ] 웹앱에서 최신 온도·습도·조도·소음 4종 표시
- [ ] 웹앱 버튼 → 시트 명령 → LED 반응
- [ ] 시트: `파일` → `다운로드` → `CSV` 로 데이터 내려받기
- [ ] 팀원 전원 폰에 PWA 설치

**이 시스템이 팀 프로젝트의 '뼈대 템플릿'이 돼요 — 소중히 보관!**

> 💡 안 되면 흐름 순서대로 점검: ① URL 직접 열기 ② 시트에 줄 생기나 ③ 시리얼 모니터 에러 확인 — **어느 구간에서 끊겼는지 찾는 게 디버깅이에요.**

### 정리 — 구조 A vs 구조 B, 언제 무엇을 쓸까

| 비교 항목 | 구조 A: ESP32 단독 (3회차) | 구조 B: 클라우드 연동 (오늘) |
|---|---|---|
| 인터넷 | 불필요 — ESP32가 WiFi 생성 | 필요 — 폰·보드 모두 |
| 웹페이지 위치 | ESP32 내부 | 구글 (AI Studio) |
| 데이터 저장 | 전원 끄면 소멸 | 시트에 영구 저장 |
| 통계·CSV | 어려움 | 쉬움 (시트가 곧 표) |
| 접속 범위 | 근거리 (같은 WiFi) | 어디서든 |
| 반응 속도 | 즉시 | 폴링 주기만큼 지연 |
| 어울리는 일 | 원리 학습 · 근거리 리모컨 | 기록·통계가 필요한 서비스 |

**우리 팀 프로젝트는 기록과 통계가 핵심 → 구조 B를 표준으로 사용해요.**

## 🔧 자주 막히는 문제 해결소 — 4회차

| 증상 | 해결 |
|---|---|
| URL을 열면 '액세스 권한이 없습니다' | 배포 설정의 액세스가 **'모든 사용자'**인지 재확인 → 수정했으면 `[배포 관리]`에서 새 버전으로 재배포 |
| 코드를 고쳤는데 반영이 안 돼요 | Apps Script는 수정 후 **반드시 재배포** — 배포 관리 → 연필 → 새 버전 → 배포 (URL이 바뀌면 ESP32 코드의 URL도 교체!) |
| ESP32가 WiFi에 연결이 안 돼요 | 핫스팟 이름·비번 오타 확인, **2.4GHz인지** 확인 (ESP32는 5GHz 불가) |
| 요청은 보냈는데 시트에 안 쌓여요 | **`setFollowRedirects`** 줄이 있는지 확인 — 구글은 요청을 한 번 '재안내'하는데 이걸 따라가야 해요 |
