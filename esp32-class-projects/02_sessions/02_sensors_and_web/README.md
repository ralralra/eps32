# 2회차 — 보드가 세상을 느끼는 법 + 나의 첫 웹페이지

> 센서로 세상을 숫자로 읽고, HTML·CSS·JS로 화면을 직접 만들어요.
> **센서 입력 + 웹 기본기 · 4교시 (50분 수업 + 10분 휴식)**

## 오늘의 목표

- [ ] 디지털 입력과 아날로그 입력의 차이를 설명하고 센서 값을 읽을 수 있다
- [ ] 센서 값에 따라 출력장치가 반응하는 '입력→판단→출력' 프로그램을 만들 수 있다
- [ ] HTML·CSS·JS의 역할을 구분하고, 버튼이 동작하는 웹페이지를 직접 작성한다

## 시간표

## 1교시 — 입력: 보드의 감각기관

1회차가 '말하기(출력)'였다면, 오늘은 '듣기(입력)'!

### 디지털 입력 vs 아날로그 입력

| | 디지털: 둘 중 하나 | 아날로그: 연속된 정도 |
|---|---|---|
| 값 | 0 또는 1, 딱 두 가지 | 0 ~ 4095 사이 숫자 |
| 의미 | 눌렸다/안 눌렸다 · 켜짐/꺼짐 | 밝기 '정도' · 시끄러운 '정도' |
| 예 | 진동센서, 홀센서(자석 감지) | 빛, 소리, 토양습도 |
| 읽기 | `digitalRead(핀)` | `analogRead(핀)` |

> 💡 왜 4095? ESP32는 전압을 4096단계(12비트)로 쪼개 읽어요 — 컴퓨터가 다룰 수 있는 건 결국 숫자!

### DSL 세트 탐험 ① 바로 읽는 센서

| 장치 | 종류 | 읽는 법 | 우리 프로젝트에서 |
|---|---|---|---|
| 빛 | 아날로그 | `analogRead(핀)` | 화분 일조량 · 환경 모니터 |
| 소리 | 아날로그 | `analogRead(핀)` | 교실 소음 · 호출 감지 |
| 토양습도 | 아날로그 | `analogRead` — **A2~A5 위치!** | 화분 핵심 (6회차 연결) |
| 진동 | 디지털 | `digitalRead(핀)` | 충격·움직임 감지 |
| 홀(자기) | 디지털 | `digitalRead` + 자석 | 문 열림 감지 (출석·안전) |
| 초음파 | 시간 측정 | `pulseIn()` → 거리 계산 | 우산 거치 · 접근 경보 |

> 🔑 아날로그 셀의 자리 규칙: **A2~A5 위치 포트(GPIO 35·34·36·39)** — WiFi와 충돌하지 않는 자리! (포트-핀맵 카드 확인)

### DSL 세트 탐험 ② 라이브러리 센서 (6회차에 팀별로 골라 써요)

| 장치 | 읽는 법(예) | 우리 프로젝트에서 |
|---|---|---|
| 온습도(DHT) | `readTemperature()` · `readHumidity()` | 화분 · 환경 모니터 (오늘 실습!) |
| 물온도 | 방수형 — 물에 직접 담가 측정 | 화분 물탱크 온도 |
| 무게 | 저울처럼 무게를 숫자로 | 매점 실물 재고 · 우산 감지 |
| 대기압 | 기압 변화 읽기 | 환경 모니터 보조(날씨 관찰) |
| CO2 | 농도 읽기 (예열 필요) | 환기 알림 — 환경 모니터 |
| 미세먼지 | 공기 중 입자 농도 | 실내 공기질 (변화 비교용) |
| 자이로 | 기울기·움직임 읽기 | 거치대 기울임·낙하 감지 |
| 심박 | 맥박 파형 읽기 | 보건·안전 체험 코너 |

> 요령: 블록코딩에 전용 블록이 있으면 **블록으로 먼저 확인 → 블록↔C 전환**으로 코드를 얻는 게 지름길.

### 따라하기 ① 진동·빛 읽기 — 시리얼 모니터로 관찰

시리얼 모니터 = 보드가 컴퓨터에 보내는 **실시간 일기장** → [`code/01_vib_light_serial/01_vib_light_serial.ino`](code/01_vib_light_serial/01_vib_light_serial.ino)

1. 우상단 돋보기 아이콘(또는 `툴` → `시리얼 모니터`) 클릭
2. 우하단 속도를 **115200 baud**로 — 코드의 `Serial.begin(115200)`과 같아야 글자가 안 깨져요
3. 빛센서를 손으로 가렸다 떼며 숫자 관찰 (0~4095 오르내림)
4. 책상을 두드리며 진동 값 0↔1 확인
5. '자동 스크롤' 체크

> ✓ 글자가 외계어로 깨져 보이면 속도(baud) 불일치 — 115200으로 맞추면 해결. **이 '관찰' 습관이 디버깅의 절반!**

### 🎯 미션 ① 빛 반응 무드등

**목표**: 교실이 어두워질수록 내장 LED가 밝아져요 (밝기 값을 LED 밝기로 '반대로' 번역)

완성 조건:
- [ ] 시리얼 모니터로 빛센서 값의 범위 확인 (가장 밝을 때 · 손으로 가렸을 때)
- [ ] `map()` 함수로 0~4095 → 255~0 변환 (반전!)
- [ ] 센서를 가리면 LED가 밝아지고, 떼면 어두워짐

> 💡 힌트: `int b = map(l, 0, 4095, 255, 0); analogWrite(LED, b);` — map은 '범위 번역기'. 뒤 두 숫자를 바꾸면 방향이 반전돼요. 예시: [`code/02_mood_light/02_mood_light.ino`](code/02_mood_light/02_mood_light.ino)

## 2교시 — 온습도·초음파 + 판단하는 보드

읽기만 하면 관찰, 읽고 **판단**하면 '스마트'가 돼요.

### 온습도 센서(DHT) — 교실의 공기를 숫자로

화분 프로젝트·환경 모니터의 핵심 센서 → [`code/03_dht/03_dht.ino`](code/03_dht/03_dht.ino)

- `float` = 소수점 있는 숫자 (25.3 같은)
- `readTemperature()` / `readHumidity()` 두 함수면 끝
- ⚠ DHT11은 **2초에 1번만** 측정 가능 — delay 필수
- 값이 `nan`이면? 배선·핀번호·측정 간격 확인
- 재미 포인트: 센서에 입김 불기 → 습도 급상승!

### 초음파 센서 — 박쥐처럼 거리 재기

소리를 쏘고, 메아리가 돌아오는 시간으로 거리를 계산 → [`code/04_ultrasonic/04_ultrasonic.ino`](code/04_ultrasonic/04_ultrasonic.ino)

- 소리 속도 초속 340m → 0.034cm/㎲, 왕복이므로 ÷2
- `readDistance()`처럼 '내 함수'를 만들면 재사용 가능
- 우산 대여함의 '우산 있음/없음' 감지가 바로 이 원리!

### 조건문 if — 보드에게 '판단'을 가르치다

**입력 → 판단 → 출력**: 모든 스마트 기기의 뼈대

```cpp
if (d < 10) {        // 10cm 이내? (참일 때)
  // WARNING 표시 + LED 켜기
} else {             // 아니라면 (거짓일 때)
  // SAFE 표시 + LED 끄기
}
```

- 블록코딩의 `[만약~라면]` 블록과 완전히 같아요
- 숫자 10을 바꾸면 '민감도'가 바뀌어요 — 직접 실험!

### 🎯 미션 ② 가까이 오면 경보!

**목표**: 물체가 10cm 안으로 접근하면 LCD에 `WARNING!!` + LED 점멸, 멀어지면 `SAFE`

완성 조건:
- [ ] 초음파 거리 측정이 시리얼에 정상 출력
- [ ] 10cm 기준으로 경보 동작
- [ ] 심화: 거리에 따라 3단계 (SAFE / CAREFUL·느린 점멸 / WARNING·빠른 점멸)

> 💡 힌트: 3단계는 `if ~ else if ~ else` 구조 — 조건을 위에서부터 차례로 검사해요. 예시: [`code/05_proximity_alarm/05_proximity_alarm.ino`](code/05_proximity_alarm/05_proximity_alarm.ino)

## 3교시 — 웹의 기본기: HTML과 CSS

AI에게 시키기 전에, 딱 한 번은 **내 손으로** 만들어 봐요.

### 웹페이지는 세 가지 재료로 만들어요

| 재료 | 역할 | 비유 |
|---|---|---|
| **HTML** | '여기 제목, 여기 버튼이 있다' — 내용과 구조 | 골격 |
| **CSS** | '제목은 파랗게, 버튼은 크고 둥글게' — 모양과 색 | 옷차림 |
| **JavaScript** | '버튼을 누르면 이렇게 해라' — 행동과 반응 | 행동 |

오늘 셋을 모두 만나고, 5회차에 AI(AI Studio)가 이걸 대신 써 주는 걸 보게 돼요 — **읽을 줄 알면 고칠 수 있어요!**

### 따라하기 ② 나의 첫 HTML

메모장(또는 에디터)에 입력 → `리모컨.html`로 저장 → 더블클릭으로 브라우저에서 열기
→ [`code/06_remote_step1.html`](code/06_remote_step1.html)

- `<태그>내용</태그>` — 여는 태그와 닫는 태그가 한 쌍
- `h1`=큰 제목, `p`=문단, `button`=버튼
- `charset="utf-8"` 이 있어야 한글이 안 깨져요
- 저장 후 새로고침(F5)하면 바뀐 게 바로 보여요

**저장 방법이 절반!**
1. 메모장: `파일` → `다른 이름으로 저장`
2. 파일 이름: `리모컨.html` — 메모장은 '파일 형식: **모든 파일**'로 바꿔야 .txt가 안 붙어요
3. 인코딩: **UTF-8** 선택 (한글 깨짐 방지 핵심!)
4. 저장 위치: 바탕화면의 팀 폴더 (다음 주에 또 씀!)
5. 이후 루틴: **수정 → Ctrl+S → 브라우저 F5**

> ✓ 주소창이 `C:\...`로 시작하면 정상 — 아직 서버가 아니라 '내 컴퓨터 파일'을 보는 중 (3회차 복선!)

### 따라하기 ③ CSS로 꾸미기

`<head>` 안에 `<style>`을 추가 — 모바일에서 누르기 좋게 '크고 둥글게'
→ [`code/07_remote_step2_css.html`](code/07_remote_step2_css.html)

- `선택자 { 속성: 값; }` 구조
- `#2E6BE6` 같은 코드가 '색상 번호'(RGB)
- `padding` = 버튼 안쪽 여백 → 버튼이 커져요
- `border-radius` = 모서리 둥글기
- 우리 과정 공통 규칙: **파란 계열 · 큰 버튼 · 모바일 우선**

## 4교시 — JavaScript: 버튼에 생명을

### 따라하기 ④ 클릭 이벤트

`</body>` 바로 위에 `<script>` 추가 → [`code/08_remote_step3_js.html`](code/08_remote_step3_js.html)

- `onclick="함수()"`: 클릭하면 이 함수를 실행하라
- `id`는 요소의 이름표 — JS가 이름표로 요소를 찾아요
- `innerText` = 글자 내용을 바꿔요
- ⚠ 아직 진짜 LED는 안 켜져요 — 화면 글자만! 그럼 진짜 LED는 언제? ↓

### F12 — 웹 개발자의 청진기

버튼이 말을 안 들을 때, 브라우저가 이유를 말해주고 있어요.

1. 브라우저에서 **F12** (또는 우클릭 → 검사)
2. 상단 탭에서 **Console** 클릭 — 에러 메시지가 표시되는 곳
3. 버튼을 눌러 보고 빨간 글씨 확인 — 에러가 없으면 조용해요(조용하면 JS는 무죄)
4. 에러 문장 읽기: `ledOn is not defined` = 'ledOn을 못 찾겠어' → 철자 확인!
5. 고치고 저장(Ctrl+S) → F5 → 재확인 — 이 반복이 웹 개발의 기본 리듬

> ✓ 에러 문장은 '범인의 자백' — 무섭다고 안 읽으면 영원히 못 잡아요.

### fetch — 다음 주를 여는 열쇠

JS가 '다른 컴퓨터에게 부탁을 보내는' 한 줄: `fetch("주소")`

```
버튼 클릭 → fetch("/led/on") 실행 → ??? (다음 주 공개) → 진짜 LED가 켜져요!
```

오늘 페이지의 버튼은 '말은 하는데 들어줄 상대가 없는' 상태.
**3회차: ESP32가 그 '상대(서버)'가 되어줘요.**

### 🎯 미션 ③ 내 리모컨 웹페이지 완성

**목표**: 제목 + 상태 표시 + 크고 둥근 버튼 2개(켜기/끄기), 누르면 상태 글자가 바뀌는 페이지

완성 조건:
- [ ] HTML: h1 제목, 상태 p(id 부여), 버튼 2개
- [ ] CSS: 파란 계열, 버튼 크게(모바일 기준), 가운데 정렬
- [ ] JS: 클릭 시 상태 글자 변경
- [ ] 팀 이름과 개성을 담을 것 — **다음 주 이 페이지가 ESP32에 들어가요**

> 💡 막히면 순서대로: ① 브라우저 F5 했는지 ② 따옴표·괄호 짝 맞는지 ③ id 철자가 같은지

## 🔧 자주 막히는 문제 해결소 — 2회차

| 증상 | 해결 |
|---|---|
| 한글이 ???로 깨져 보여요 | `<meta charset="utf-8">` 확인 + 저장할 때 인코딩 UTF-8 선택 |
| 센서 값이 nan 또는 0만 나와요 | 핀 번호를 핀맵과 대조, DHT는 측정 간격 2초 이상, 배선 재확인 |
| 버튼을 눌러도 아무 반응이 없어요 | F12 → Console 탭의 빨간 에러 읽기 — 함수 이름·id 철자 불일치가 대부분 |
| CSS가 적용이 안 돼요 | `<style>`이 `<head>` 안에 있는지, 중괄호 짝, 세미콜론 확인 |

- if 조건문으로 '판단하는 보드'(접근 경보기)를 만들었어요
- HTML·CSS·JS의 역할을 구분하며 첫 웹페이지를 직접 작성했어요
- 클릭 이벤트로 화면이 반응하게 만들었어요
