# 발표 구성 템플릿 — 5단계 스토리 (발표 5분 + 시연 4분 + 질의 2분)

> 기술 자랑이 아니라 **'문제 해결 이야기'**를 들려줘요.

## ① 문제 정의 (1장 · ~1분)

- 우리 학교의 어떤 불편함에서 출발했나 — **사진 한 장**이면 최고
- 예: "쉬는 시간 매점 줄이 이렇게 깁니다" / "비 오는 날 우산꽂이가 이렇게 됩니다"

```
우리가 발견한 문제:
그래서 우리는 ______를 만들기로 했다.
```

## ② 우리의 설계 (1~2장 · ~1.5분)

- 시스템 구조도: 웹앱 → Apps Script → 시트 → ESP32 그림
- **구조 B(클라우드)를 선택한 이유** — 4회차 비교표가 근거!
- 시트 설계(탭·열)를 한 장으로

## ③ 시연 (실물 · 4분) — 발표의 심장

- 핵심 시나리오를 **폰 → 장치 → 시트** 순서로
- 역할: 조작 1명 · 해설 1명 (리허설로 확정)
- 💾 시연이 멈추면: 침착하게 "준비한 영상으로 보여드리겠습니다" — 이것도 실력!

## ④ 우회와 배움 (1장 · ~1분)

- '~대신 ~를 쓴 이유' — **제약을 설계로 푼 이야기는 감점이 아니라 가점!**
- 예: "RFID 리더가 없어서 웹 QR 방식으로 바꿨는데, 오히려 폰만 있으면 돼서 더 편해졌습니다"
- 교차 테스트에서 발견된 한계 + 개선 계획도 여기서

## ⑤ 데이터와 확장 (1장 · ~1분)

- 쌓인 데이터·통계 보여주기 (시트/CSV/통계 화면)
- "한 달 더 있다면?" 확장 계획 한 가지

## 준비 체크리스트

- [ ] 리허설 2회 완료 (1회는 다른 팀원 앞에서, 시간 실측!)
- [ ] 백업 영상 촬영 — 폰과 노트북 양쪽 저장, 재생 테스트까지
- [ ] 핫스팟·보드 전원·화면 미러링 점검
- [ ] 부스 세팅 사진 촬영 (본 발표 세팅 3분 컷)
