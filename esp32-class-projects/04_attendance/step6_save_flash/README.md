# 6단계 — ESP32 안에 저장하기 (전원 꺼도 유지)

## 목표
출석 기록을 ESP32 **내부 저장공간(SPIFFS)** 에 적어둔다. 전원을 껐다 켜도 남는다.

## 핵심 아이디어 — 3단계가 여기서 다시!
3단계에서 브라우저의 `localStorage` 에 저장했죠.
ESP32에도 똑같이 작은 저장공간이 있어요. 이름이 **SPIFFS** 예요.

| | 3단계 (브라우저) | 6단계 (ESP32) |
|---|---|---|
| 저장 장소 | localStorage | SPIFFS (내부 플래시) |
| 저장 방식 | 꺼내기→추가→넣기 | 파일 끝에 한 줄 추가 |
| 전원 꺼도 유지 | ✅ | ✅ |
| 누가 볼 수 있나 | 그 브라우저만 | **접속한 모두** (공용 출석부!) |

→ **개념은 똑같고**, 저장 장소만 보드 안으로 바뀐 거예요.

## 시각 문제와 해결 (중요)
ESP32는 인터넷이 없어서 **지금이 몇 시인지 모릅니다** (시계가 없음).
그래서 똑똑한 방법을 씁니다: **학생 폰의 시각을 같이 보내기.**

```javascript
var time = new Date().toLocaleString();   // 폰이 알고 있는 현재 시각
fetch("/check?name=" + name + "&time=" + time)
```

## 코드에서 새로 생긴 것

```cpp
#include <SPIFFS.h>            // 내부 저장공간 사용

SPIFFS.begin(true);            // 저장공간 준비 (setup에서)

File f = SPIFFS.open("/attend.csv", FILE_APPEND);  // 파일 열기(끝에 추가 모드)
f.println(name + "," + time);  // 한 줄 쓰기:  홍길동,2026-06-28 14:20:00
f.close();                     // 닫기
```

> `FILE_APPEND` = 기존 내용 뒤에 이어 쓰기 (덮어쓰지 않음)
> `.csv` = 쉼표로 칸을 나눈 표 파일. 엑셀로 바로 열려요.

## 보드 설정 확인
`Tools` → `Partition Scheme` → **Default 4MB with spiffs** (SPIFFS 공간 필요)

## 해보기
1. 업로드 → 시리얼 모니터 열기
2. 폰으로 접속해서 이름 2~3명 출석
3. 시리얼에 `출석 저장: 홍길동 (2026-...)` 찍히는지 확인
4. **진짜 시험**: ESP32의 USB를 뽑았다가 다시 꽂기 (전원 재시작)
5. (7단계 페이지로 확인하거나) 다시 출석 몇 개 더 → 이전 기록 위에 계속 쌓임

> 6단계에서는 저장된 걸 보는 화면이 아직 없어요. "저장이 되는지"는
> 시리얼 메시지로 확인하고, **전체 명단 보기는 7단계**에서 만듭니다.

## 확인 ✅
- [ ] 출석하면 시리얼에 "출석 저장: ..." 이 찍힌다
- [ ] 폰에 시각이 포함된 답이 온다
- [ ] 전원을 껐다 켜도 (7단계에서 확인) 기록이 남아있다

## 배운 것
- ESP32 내부 저장공간 = SPIFFS
- 파일에 한 줄씩 이어 쓰기 (`FILE_APPEND`)
- 시계 없는 ESP32는 폰의 시각을 빌려 쓴다
- CSV = 엑셀로 열리는 표 파일
