# 1단계 — 키오스크 화면 만들기 (브라우저에서)

## 목표
상품을 골라 장바구니에 담고, 가상 포인트로 **모의결제**하는 키오스크 화면을 브라우저에서 먼저 만든다.

## 왜 브라우저 먼저?
출석 프로젝트와 같은 방식이에요. 화면·버튼·결제 흐름을 **ESP32 없이 눈으로** 먼저 완성하면, 나중에 ESP32로 옮기기가 쉬워요.

## 해보기
1. `index.html` 을 브라우저로 열기
2. 상품 버튼을 눌러 장바구니에 담기
3. "결제하기" → 포인트가 차감되고 "결제 완료!" 메시지
4. 포인트보다 비싸게 담으면 → "포인트가 부족합니다!"
5. 새로고침해도 남은 포인트가 유지되는지 (localStorage 저장)

## 확인 ✅
- [ ] 상품을 누르면 장바구니에 쌓이고 합계가 오른다
- [ ] 결제하면 포인트가 줄어든다
- [ ] 포인트 부족 시 결제가 막힌다
- [ ] 새로고침해도 남은 포인트가 유지된다

## 코드 핵심 — 모의결제
```javascript
if (points < total) { 부족 처리; return; }
setPoints(points - total);   // 포인트에서 결제금액 차감
```
"결제"란 결국 **잔액에서 금액을 빼는 것**이라는 걸 체험해요. (실제 돈/카드사 연동 없이 흐름만)

## 직접 바꿔보기
- 상품 목록(`products`)에 과자 추가
- 시작 포인트를 바꾸기 (localStorage 초기값 10000)
- 장바구니에서 항목 삭제 기능 만들기 (도전)

## 배운 것
- 키오스크 = 상품 선택 → 장바구니 → 결제 흐름
- 모의결제 = 잔액에서 금액 차감 (창업/결제 개념)
- 출석 프로젝트의 "브라우저 먼저" 방식 재사용
