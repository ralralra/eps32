# 4단계 — 폰으로 수분값 보기 (웹 대시보드)

## 목표
폰으로 접속하면 **현재 수분값과 상태가 실시간으로 보이고**, "지금 물주기" 버튼으로 수동 급수도 한다.
자동 급수(3단계)도 그대로 동작.

## 핵심 — 출석 프로젝트가 여기서 재활용!
출석에서 배운 그 **AP 모드 웹서버**를 그대로 가져왔어요. 화면 내용만 "출석" → "수분값"으로 바뀐 거예요.
```cpp
WiFi.softAP(AP_SSID, AP_PASS);   // ESP32가 와이파이 만들기 (출석과 동일!)
server.on("/", handleRoot);      // 페이지 보여주기 (동일!)
```
→ 한 번 배운 기술이 다른 프로젝트에서 그대로 쓰인다는 걸 체감하는 단계예요.

## 새로 보는 것
```cpp
<meta http-equiv='refresh' content='2'>   // 페이지가 2초마다 자동 새로고침
int raw = analogRead(SOIL_PIN);            // 접속할 때마다 현재 수분값 읽기
```
- 자동 새로고침으로 **실시간 모니터링**처럼 보임
- `/water` 주소 = 수동 급수 버튼

## 해보기
1. `DRY_THRESHOLD` 를 내 센서 기준값으로 수정
2. 업로드 → 시리얼에 `접속: http://192.168.4.1` 확인
3. 폰 와이파이 `ESP32-Plant` 접속 (비번 `plant1234`)
4. 브라우저에서 **http://192.168.4.1**
5. 수분값 숫자와 "말랐어요/촉촉해요"가 2초마다 갱신되는지 확인
6. 센서를 적셨다 말렸다 하면 화면 색과 상태가 바뀜
7. "💦 지금 물주기" 버튼 → 펌프가 도는지 확인

## 확인 ✅
- [ ] 폰에서 수분값이 실시간으로 보인다
- [ ] 마름/촉촉에 따라 색과 글씨가 바뀐다
- [ ] "지금 물주기" 버튼으로 수동 급수가 된다
- [ ] 마른 상태로 두면 자동 급수도 동작한다 (시리얼 `자동 급수`)

## 잘 안 될 때
- 접속 안 됨 → 출석 프로젝트와 동일. `http://` (https X), Brownout이면 IO0+RESET, 자세히는 출석의 [`docs/troubleshooting.md`](../../04_attendance/docs/troubleshooting.md)
- **수분값이 항상 0 또는 4095 고정** → 센서가 GPIO34에 제대로 연결됐는지, ADC1 핀(32~39) 맞는지 확인

## 직접 바꿔보기
- 새로고침 주기를 2초 → 5초 (`content='5'`)
- 화면에 막대그래프나 물방울 이모지 추가
- 마름 기준을 화면에서 바꾸는 슬라이더 (도전 과제)

## 배운 것
- 한 번 배운 웹서버 기술이 **다른 프로젝트에 그대로 재사용**된다
- 센서값을 웹으로 실시간 표시 = IoT 대시보드의 기본
- 자동(규칙) + 수동(버튼) 제어를 함께 제공

## 완성! 🎉 그리고 확장 아이디어
- 수분값을 시간별로 기록해서 그래프 (SPIFFS에 CSV — 출석 6단계 방식)
- DHT11 온습도 센서 추가
- 물탱크 수위 센서 (물 부족 알림)
- 여러 화분 동시 관리
