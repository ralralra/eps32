# AI Studio로 웹앱 만들기 — New App부터 폰 설치(PWA)까지

> 2회차에 손으로 쓰던 HTML·CSS·JS를 AI가 대신 써 줘요 — **우리는 읽고 고칠 줄 알아요!**
> 이 문서 하나로: **New App 만들기 → 프롬프트 샘플 → 다듬기 → 링크 공유 → 모두의 폰에 설치**까지.
> 수업 흐름 속 위치: [4회차 3교시](../02_sessions/04_cloud_expansion/README.md) · [5회차 3~4교시](../02_sessions/05_team_project_planning/README.md)

## 1. New App 만들기 (최초 진입)

1. 브라우저에서 **aistudio.google.com** 접속 → **팀 구글 계정**으로 로그인
2. **Build** 메뉴 → **New App** (새 앱 만들기) 선택
   - 버튼 이름·위치는 업데이트로 달라질 수 있어요 — 'Build'·'New app' 계열을 찾자
3. 프롬프트 입력 칸에 아래 샘플을 붙여넣고 `[ ]` 부분만 우리 팀 것으로 교체
4. 생성 대기 → 미리보기에서 버튼을 직접 눌러보기

> ✏ AI 활용의 핵심 = **요구사항을 명확한 문장으로 쓰는 능력.**
> "매점 앱 이쁘게" ❌ → 화면 구성·디자인 규칙·동작 규칙의 **3단 구조** ⭕

## 2. 프롬프트 지시문 샘플

### 기본형 — 교실 환경 모니터 (4회차, 전 팀 공통)

```
교실 환경 모니터 웹앱을 만들어줘.

화면 구성 :
- 상단 제목 : "[1팀] 교실 환경 모니터"
- 큰 카드 4개 : 온도, 습도, 조도, 소음
- 카드마다 큰 숫자와 단위 표시
- 하단 : LED 켜기 / LED 끄기 버튼 2개

디자인 규칙 :
- 파란 계열, 모바일 우선, 버튼 크게, 글자 크게

동작 규칙 :
- 10초마다 [우리 팀 웹앱 URL]?mode=latest 에서
  최신 값을 가져와 카드에 표시해줘
- LED 버튼을 누르면
  [우리 팀 웹앱 URL]?mode=set&cmd=ON (또는 OFF)
  으로 요청을 보내줘
```

### 팀 프로젝트형 — 매점팀 원문 (5회차, 다른 팀은 이름만 교체)

```
학교 매점 관리 웹앱(PWA)을 만들어줘.

화면 1 · 판매 화면 :
- 제목 "[3팀] 스마트 매점"
- 상품 카드 6개 : 이름, 가격, 남은 재고, [판매] 버튼
- [판매]를 누르면
  [웹앱 URL]?action=sell&item=상품명 으로 요청하고
  "판매 완료" 메시지를 잠깐 보여줘

화면 2 · 통계 화면 :
- 오늘 판매 수와 인기 상품 순위 표시
- [웹앱 URL]?action=stats 에서 가져와 표시
- 화면 1과 탭으로 전환되게 해줘

공통 디자인 : 파란 계열, 모바일 우선, 버튼 크게, 글자 크게
```

### 팀별 교체 가이드

| 팀 | 화면 1 | 화면 2 | action 예시 |
|---|---|---|---|
| ① 매점 | 상품 카드 + [판매] | 판매 수·인기 순위 | `sell`, `stats` |
| ② 키오스크 | 메뉴 + 장바구니 + [주문] | 주문번호·매출 통계 | `order`, `sales` |
| ③ 우산 | QR 스캔 + [대여]/[반납] | 대여 현황·미반납 | `rent`, `return`, `status` |
| ④ 화분 | 현재 수분·온습도 카드 | 기록 그래프·급수로그 | `latest`, `history`, `water` |
| ⑤ 출석 | QR 체크인 + 현재 인원 | 출석부·통계 | `checkin`, `checkout`, `list` |

### 추가 지시문 모음 (필요할 때 한 문장씩)

| 원하는 것 | 프롬프트 한 문장 |
|---|---|
| QR 스캔 (우산·출석팀) | "카메라로 QR을 스캔하는 기능을 추가해줘" (html5-qrcode 방식) |
| 알림 소리 | "판매/체크인이 완료되면 알림 소리를 재생해줘" (부저 없이 폰이 스피커!) |
| 바깥 날씨 카드 | "Open-Meteo API(키 필요 없는 무료 날씨 API)로 위도 [37.57] 경도 [126.98]의 현재 기온·습도를 '바깥 날씨' 카드로 추가해줘. 10분마다 갱신." |
| 화면 다듬기 | "온도 카드를 더 크게" / "제목을 [1팀] 모니터로" — **한 번에 하나씩!** |
| 기능 빼기 | "로그인 기능은 빼줘" — 범위 지키기는 여기서도 계속! |

> ⚠ 이름 일치 주의: `action`·파라미터 철자는 Apps Script 분기·시트와 **완전히 같아야** 해요 → [4회차 '이름 대조표'](../02_sessions/04_cloud_expansion/README.md)
> ⚠ URL 붙여넣기 전 끝이 **`/exec`** 인지 확인 (`/dev`는 동작 안 함)

## 3. 생성 후 — AI는 초안, 검토는 사람

1. **스케치와 비교**하며 다른 점 목록 만들기 → 하나씩 추가 지시로 해소
2. **코드 보기**로 HTML·CSS·JS 구조 확인 — 2회차에 배운 3형제 그대로! id 이름을 기록해 두면 연결이 편해요
3. 화면 전환·버튼을 직접 눌러보기 — 동작 안 하는 버튼은 '아직 껍데기'
4. fetch 주소를 **우리 팀 웹앱 URL**로 연결: "fetch로 [URL]에서 데이터를 가져와 표시해줘"

## 4. 링크 공유 → 모두의 핸드폰에서 보기 (PWA 설치)

**링크로 열고 홈 화면에 추가하면 아이콘이 생겨요 — 이것이 PWA(설치형 웹앱)!**

1. AI Studio에서 **공유/배포(Share) 버튼** → 앱 링크 만들기
2. 링크를 팀 채널로 전송 — 또는 검색창에 'QR 생성기' → 링크 붙여넣기 → **QR을 교실 화면에 띄우면 모두 스캔!**
3. 폰 브라우저로 링크 열기 → 카드·버튼이 정상 표시되는지 확인
4. **홈 화면에 추가**:
   - 안드로이드: 메뉴(⋮) → **'홈 화면에 추가'**
   - 아이폰: 공유(⬆) → **'홈 화면에 추가'**
5. 홈 화면 아이콘 터치로 실행 — 주소창 없는 전체 화면, 겉보기엔 완전한 앱! 📱

> ✓ **팀원 전원 폰에 설치**가 미션 완성 조건 (4·5회차 체크리스트).
> ✓ 웹앱을 수정해도 링크가 같으면 재설치 불필요 — 폰에서 새로고침만 하면 최신 화면.
> ✓ 발표(7회차) 때도 이 링크로 청중 폰에서 바로 시연할 수 있어요.

## 5. 자주 막히는 문제

| 증상 | 해결 |
|---|---|
| 생성 결과가 우리 스케치와 달라요 | 정답은 'AI 화면'이 아니라 '우리 스케치' — 차이 목록을 만들어 **한 번에 하나씩** 수정 지시 |
| 버튼을 눌러도 데이터가 안 와요 | fetch 주소 확인: 끝이 `/exec`인지, action 철자가 Apps Script와 같은지 |
| 폰에서 화면이 깨져요 | "모바일 우선으로, 버튼과 글자를 더 크게 만들어줘" 재지시 |
| 홈 화면 추가 메뉴가 안 보여요 | 아이폰은 Safari, 안드로이드는 Chrome에서 여는 게 가장 확실 |
