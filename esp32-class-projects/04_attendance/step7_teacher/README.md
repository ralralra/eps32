# 7단계 — 선생님용 전체 출석부 (완성본) 🎉

## 목표
선생님이 `/list` 주소로 접속하면 **전체 출석 명단을 표로** 보고,
**CSV로 다운로드** 하거나 **전체 초기화** 할 수 있다.

이 단계가 출석 프로젝트의 **완성본** 입니다.

## 두 개의 화면
```
학생 화면   →  http://192.168.4.1/        이름 넣고 출석
선생님 화면 →  http://192.168.4.1/list    전체 명단 + 다운로드 + 초기화
```
같은 ESP32가 주소(`/` vs `/list`)에 따라 다른 화면을 보여줍니다.

## 코드에서 새로 생긴 것 — 주소(기능)가 늘었다

```cpp
server.on("/",         handleRoot);     // 학생 출석 화면
server.on("/check",    handleCheck);    // 출석 저장
server.on("/list",     handleList);     // ★ 선생님: 전체 명단 표
server.on("/download", handleDownload); // ★ 선생님: CSV 내려받기
server.on("/reset",    handleReset);    // ★ 선생님: 초기화
```

명단 만들기 = 저장된 파일을 **한 줄씩 읽어서** 표로 그리기:
```cpp
File f = SPIFFS.open(FILE_PATH, FILE_READ);
while (f.available()) {
  String line = f.readStringUntil('\n');   // 한 줄 = "홍길동,2026-..."
  int comma = line.indexOf(',');           // 쉼표 위치 찾기
  String name = line.substring(0, comma);  // 쉼표 앞 = 이름
  String time = line.substring(comma + 1); // 쉼표 뒤 = 시각
  // → 표의 한 행으로 추가
}
```
> 6단계에서 `이름,시각` 으로 저장했으니, 여기선 **쉼표로 잘라서** 다시 나눕니다.

## 해보기
1. 업로드 → 시리얼에 두 주소가 뜨는지 확인
2. 폰으로 학생 출석 몇 명 하기 (`/`)
3. 주소창에 **192.168.4.1/list** 입력 (또는 출석화면 맨 아래 링크)
4. 출석한 사람들이 표로 보이는지 확인
5. **📥 CSV 다운로드** → 파일이 받아지는지 (PC면 엑셀로 열림)
6. **🗑 전체 초기화** → 확인창 뜨고, 누르면 명단이 비워짐
7. **전원 재시작 테스트**: USB 뽑았다 꽂기 → /list 다시 보기 → 기록이 남아있으면 6단계 저장이 진짜로 된 것! ✅

## 확인 ✅
- [ ] /list 에서 전체 명단이 표로 보인다
- [ ] "총 N명 출석" 숫자가 맞다
- [ ] CSV 다운로드가 된다
- [ ] 초기화하면 명단이 비워진다
- [ ] 전원 껐다 켜도 명단이 남아있다 (← 진짜 저장 확인!)

## 수업 운영 팁
- 선생님은 `/list` 를 띄워두고 실시간으로 출석 현황 확인
- 수업 끝나면 CSV 다운로드해서 보관 → 다음 시간 전에 초기화
- 반이 여러 개면 보드마다 `AP_SSID` 를 `ESP32-1반`, `ESP32-2반` 으로

## 더 해보기 (확장 아이디어)
- 같은 이름 중복 출석 막기 (이미 있으면 "이미 출석함" 표시)
- 출석 순서대로 번호 매기기 (이미 됨!) → 지각 판단에 활용
- 인터넷 되는 환경이면 Google Sheets로 전송해서 클라우드 저장
- RFID 카드로 업그레이드 (대리출석 방지)

## 배운 것 (출석 프로젝트 전체 정리)
1. HTML로 화면 만들기 (1단계)
2. 버튼에 동작 연결하기 (2단계)
3. 저장 개념 (3단계, 브라우저)
4. ESP32가 페이지 보내기 (4단계)
5. 폰→ESP32 데이터 전송 (5단계)
6. ESP32에 저장 (6단계)
7. 전체 조회·다운로드 (7단계)
