# 3단계 — 새로고침해도 안 사라지게 (저장 개념)

## 목표
출석 명단을 **저장** 해서, 새로고침하거나 브라우저를 껐다 켜도 그대로 남게 한다.

## 왜?
2단계의 문제: 새로고침하면 명단이 날아갔죠.
이유는 **"화면에 보여주기"** 와 **"저장하기"** 가 다른 일이기 때문이에요.
- 화면에 보여주기 = 잠깐 보이는 것
- **저장하기 = 어딘가에 적어두는 것** ← 이게 빠졌었음

이번엔 브라우저가 가진 작은 저장 공간 **localStorage** 에 적어둡니다.
(나중에 ESP32에서는 이 저장 공간이 "보드 내부 메모리"로 바뀝니다. 개념은 똑같아요!)

## 해보기
1. `index.html` 열기
2. 이름 몇 개 출석시키기
3. **새로고침(F5)** → 명단이 그대로 남아있는지 확인! 👀
4. 브라우저를 완전히 껐다가 다시 `index.html` 열기 → 여전히 남아있음
5. "전체 지우기" 누르면 깨끗이 지워짐

## 확인 ✅
- [ ] 출석 후 새로고침해도 명단이 남아있다
- [ ] 브라우저를 껐다 켜도 남아있다
- [ ] "전체 지우기"로 삭제된다

## 핵심 코드 — 저장의 3박자

```javascript
// 1) 꺼내기 — 저장소에서 기존 명단 읽기
var data = JSON.parse(localStorage.getItem("attend") || "[]");

// 2) 추가하기 — 새 출석을 명단에 더하기
data.push({ name: name, time: time });

// 3) 다시 넣기 — 저장소에 도로 저장
localStorage.setItem("attend", JSON.stringify(data));
```

**꺼내기 → 추가 → 다시 넣기.** 이 3박자가 거의 모든 "저장"의 기본 패턴이에요.

페이지가 열릴 때 저장된 걸 불러오는 부분:
```javascript
window.onload = draw;   // 페이지 열리면 저장소에서 읽어서 화면에 그림
```

## 잠깐 — localStorage가 뭐야?
브라우저마다 가진 **작은 메모장** 이에요. 글자를 적어두면 새로고침/재시작해도 남아요.
단, 이 메모장은 **그 컴퓨터의 그 브라우저 안에만** 있어요.
→ 그래서 다른 사람 폰에서는 안 보입니다. **"우리 반 전체가 함께 보는 출석부"가 되려면 한곳(ESP32)에 모아야 해요.** 그게 다음 단계!

## 배운 것
- 화면에 그리기 ≠ 저장하기
- 저장의 3박자: 꺼내기 → 추가 → 다시 넣기
- localStorage는 "그 브라우저 안에만" 저장됨 → 공용 저장소(ESP32)가 필요한 이유
