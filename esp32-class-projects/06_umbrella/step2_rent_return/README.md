# 2단계 — 대여 / 반납 처리

## 목표
**학생증 → 우산** 순서로 찍으면, 비어있는 우산은 **대여**되고 빌려간 우산은 **반납**된다.

## 원리 — 상태(state)로 기억하기
각 우산이 "누가 빌렸는지"를 숫자로 기억해요:
```cpp
int borrowedBy[] = { -1, -1, -1 };   // -1 = 비어있음, 0/1 = 그 학생이 빌림
```
- 우산을 찍었을 때 `-1`이면 → **대여** (현재 학생을 기록)
- `-1`이 아니면 → **반납** (다시 `-1`로)

이게 도서관 대출/반납과 똑같은 **토글(번갈아 전환)** 구조예요.

## 해보기
1. 학생표·우산표의 UID를 1단계에서 적은 값으로 수정
2. 업로드 → 시리얼 모니터
3. **학생증 찍기** → "학생 인증: 김OO"
4. **우산 태그 찍기** → "대여: 김OO → 우산 1번"
5. 같은 우산을 다시 찍기 → "반납: 우산 1번"
6. 학생증 없이 우산만 찍으면 → "먼저 학생증을 찍어주세요"

## 확인 ✅
- [ ] 학생 → 우산 순서로 대여가 된다
- [ ] 빌린 우산을 다시 찍으면 반납된다
- [ ] 학생증 먼저 안 찍으면 안내가 뜬다
- [ ] 다른 학생으로도 대여/반납이 된다

## 배운 것
- 상태 변수(`borrowedBy`)로 "지금 어떤 상황인지" 기억
- 같은 동작(태그)이 상태에 따라 대여/반납으로 갈린다 (토글)
- 두 종류의 카드(학생/우산)를 표로 구분
