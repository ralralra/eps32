# 6단계 — 움직임 감지

연속된 두 프레임을 비교해서 픽셀이 많이 바뀌면 "움직임이 있다"고 판단.
실제 카메라(보안/도어캠/야생동물 트랩)에서 자주 쓰는 가장 기초적인 방법.

## 아이디어
1. 카메라를 `GRAYSCALE` 모드(JPEG 아님)로 작은 해상도(QQVGA 160x120)에서 캡처
2. 다운샘플 후 직전 프레임과의 픽셀 차이를 절대값으로 합산
3. 임계값 넘으면 "motion!" 출력 + 플래시 LED 깜빡

## 왜 작은 해상도?
- 차분 계산은 RAM/CPU 부담. 작은 회색조 프레임이면 빠르고, 잡음에도 어느정도 강함
- 실제 저장은 별도 JPEG 캡처를 트리거하면 됨 (TODO로 남겨둠)

## 파일
- `motion.ino` — 가장 단순한 차분 감지

## 튜닝 포인트
| 값 | 의미 |
|---|---|
| `DIFF_THRESHOLD` (픽셀당) | 한 픽셀이 "변했다"고 볼 밝기 차이 |
| `MOTION_RATIO` | 전체 픽셀 중 변한 비율 (0.05 = 5%) |
| `COOLDOWN_MS` | 한 번 감지하면 N ms 동안 재트리거 안 함 |

## 발전 방향 (다음 프로젝트들)

- 감지 시 텔레그램/Slack 봇으로 사진 전송
- 평균 배경 모델로 조명 변화에 강하게 (running average)
