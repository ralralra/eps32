# 스마트팜 — 구글 시트 + Apps Script 만들기 (클릭 단위)

> 교실 환경 모니터링 때와 같은 방법이에요. 이름과 항목만 스마트팜용으로 바뀝니다.
> 처음이라면 4회차 자료도 참고 → [4회차 따라하기](../../esp32-class-projects/02_sessions/04_cloud_expansion/따라하기.md)

## 1. 시트 준비

1. [sheets.new](https://sheets.new) → 새 스프레드시트, 이름 `스마트팜`
2. **시트1** 탭 1행에 머리글 입력:

   | A | B | C | D | E | F |
   |---|---|---|---|---|---|
   | 시각 | 토양습도 | 온도 | 습도 | 조도 | 수위 |

3. 왼쪽 아래 **+** 로 새 탭 추가 → 이름을 **설정** 으로
4. **설정 탭 A1** = 명령 칸입니다. 일단 `AUTO` 라고 적어 두세요.
   (앱이 여기에 `PUMP_ON` 같은 명령을 쓰고, ESP32가 읽어 가요)
5. **설정 탭 B1** = 자동 급수 기준값. 일단 `1500` 입력 (토양습도가 이보다 낮으면 = 마르면 급수)

## 2. Apps Script 붙여넣기

1. 시트 메뉴 `확장 프로그램` → `Apps Script`
2. 편집기의 기존 코드 전부 지우고 → [`02_apps_script_smartfarm.gs`](../code/02_apps_script_smartfarm.gs) 내용 전체 복사 → 붙여넣기
3. 💾 저장 (Ctrl+S)

## 3. 웹앱으로 배포

1. 오른쪽 위 파란 **배포** → **새 배포**
2. ⚙(톱니) → **웹 앱** 선택
3. 설정:
   - 다음 사용자 인증: **나**
   - 액세스 권한: **모든 사용자** ← 꼭!
4. **배포** → 권한 요청이 뜨면: 계정 선택 → `고급` → `…(안전하지 않음)으로 이동` → 허용
5. 나온 **웹 앱 URL**(`https://script.google.com/macros/s/…/exec`) 복사 — **/exec로 끝나는지 확인!**

## 4. 브라우저로 먼저 테스트 (보드 없이!)

주소창에 직접 쳐 보세요 (URL 뒤에 붙이기):

| 테스트 | 주소 뒤에 붙일 것 | 기대 결과 |
|---|---|---|
| 기록 저장 | `?soil=1800&temp=25.3&humi=60&light=2000&level=3000` | 시트1에 새 줄 |
| 명령 읽기 | `?mode=cmd` | `AUTO` (설정!A1 값) |
| 명령 쓰기 | `?mode=set&cmd=PUMP_ON` | 설정!A1이 `PUMP_ON`으로 |
| 최신값 읽기 | `?mode=latest` | 마지막 줄 JSON |
| 기준값 바꾸기 | `?mode=limit&value=1600` | 설정!B1이 `1600`으로 |

다섯 개가 모두 되면 → 보드 코드([01_cloud_template](../code/01_cloud_template/01_cloud_template.ino))의 ★ 3줄만 바꿔 업로드하세요.

## 명령 규칙 (앱 ↔ 보드 약속)

| 설정!A1 값 | ESP32 동작 |
|---|---|
| `AUTO` | 자동 모드 — 기준값(B1)대로 알아서 급수·환기 |
| `PUMP_ON` / `PUMP_OFF` | 펌프 강제 켜기/끄기 (수동) |
| `FAN_ON` / `FAN_OFF` | 팬 강제 켜기/끄기 |
| `LED_ON` / `LED_OFF` | 조명 켜기/끄기 |

> ⚠ 펌프 안전 규칙: `PUMP_ON`이어도 코드가 **최대 5초만 켜고 자동으로 꺼요** (물 넘침 방지).
> 수위가 기준 이하면 펌프는 절대 켜지 않고 시트에 "물 부족"을 기록합니다.
