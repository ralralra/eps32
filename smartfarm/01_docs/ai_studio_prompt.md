# 스마트팜 앱 — AI Studio 생성 프롬프트 (가드레일 포함)

> AI Studio(aistudio.google.com) → **Build** → **New App** → 아래 프롬프트 붙여넣기.
> 만들어진 앱은 폰 브라우저에서 열어 **홈 화면에 추가**하면 앱처럼 설치돼요.
> 자세한 과정은 → [AI Studio 웹앱 가이드](../../esp32-class-projects/01_docs/ai_studio_webapp_guide.md)
>
> 키트 강의자료의 앱인벤터 앱 4종(토양습도 측정 · 온습도 측정 · LED 제어 · 쿨링팬 제어)의
> 기능이 이 앱 하나에 모두 들어 있어요 — 블루투스 연결 과정은 필요 없습니다!

## 붙여넣을 프롬프트

> `여기에_내_웹앱_URL` 을 **내 Apps Script /exec 주소**로 바꾼 뒤 붙여넣으세요!

```
스마트팜 관리 앱을 만들어줘.

[데이터 연결 — 반드시 지켜야 할 규칙]
- 서버 주소는 오직 이것 하나만 사용해: 여기에_내_웹앱_URL
- 절대 다른 서버 주소나 API를 새로 만들거나 추측하지 마.
- 데이터 읽기: GET 요청 "여기에_내_웹앱_URL?mode=latest"
  응답 JSON 형식(이 이름 그대로 사용, 바꾸지 마):
  {"time":"07-24 14:30:00","soil":45,"temp":25.3,"humi":60,"limit":30,"cmd":"AUTO"}
- 제어 명령: GET 요청 "?mode=set&cmd=명령" — 명령은 다음 5개만 존재해:
  AUTO, FAN_ON, FAN_OFF, LED_ON, LED_OFF
- 급수 알림 기준값 변경: GET 요청 "?mode=limit&value=숫자"

[화면 구성]
1. 상단: "우리 반 스마트팜 🌱" 제목과 마지막 측정 시각(time)
2. 센서 카드 3개: 토양습도(soil, % 단위), 온도(temp, ℃), 습도(humi, %)
   - 10초마다 자동 새로고침(mode=latest 폴링)
   - soil이 limit보다 낮으면 토양습도 카드를 주황색으로 강조하고 "💧 물 주세요!" 크게 표시
   - temp가 26보다 높으면 온도 카드에 "더워요 — 팬을 확인하세요" 표시
   - temp가 21보다 낮으면 온도 카드에 "추워요 — 보온등을 확인하세요" 표시
3. 제어 버튼: [🌀 팬 켜기(FAN_ON)] [팬 끄기(FAN_OFF)] [💡 보온등 켜기(LED_ON)] [보온등 끄기(LED_OFF)] [🤖 자동모드(AUTO)]
   - 현재 모드(cmd)를 화면에 표시하고, AUTO일 때는 수동 버튼 위에 "자동모드 해제 후 사용" 안내
4. 설정: 급수 알림 기준(limit) 슬라이더(0~100%) + [저장] 버튼 → mode=limit 호출
5. 모바일 화면 기준으로 크고 누르기 쉬운 버튼, 초록색 계열의 밝은 디자인

[기타]
- 한국어 UI로 만들어줘.
- 요청이 실패하면 "연결 확인 중…"이라고 표시하고 10초 뒤 다시 시도해.
```

## 왜 "규칙" 부분이 중요한가요?

AI가 앱을 만들다가 **자기 마음대로 서버 주소나 데이터 이름을 지어내는 일**이 있어요
(교실 모니터링 때 `run.app` 주소를 만들어냈던 그 현상!).
위 규칙 블록이 그걸 막아줍니다. 앱이 이상한 주소를 쓰면 이 규칙 부분을 다시 붙여넣으며 "규칙대로 고쳐줘"라고 하세요.

## 완성 후 점검 순서

1. 앱에 토양습도·온도·습도 숫자가 떠요? → 안 뜨면 브라우저에서 `URL?mode=latest` 직접 열어 JSON 확인
2. [팬 켜기] → 시트 **설정!A1**이 `FAN_ON`으로 바뀌고 → 10초 안에 팬이 돌아요?
3. 슬라이더 저장 → **설정!B1** 숫자가 바뀌어요?
4. 화분에서 센서를 뽑아 보세요 → 토양습도가 뚝 떨어지며 "💧 물 주세요!"가 떠요? (키트 자료의 확인 방법 그대로!)
5. 다 되면 폰 브라우저 메뉴 → **홈 화면에 추가** → 아이콘으로 실행!
