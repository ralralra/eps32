# 🐣 4회차 왕초보 가이드 — 구글 시트에 저장 + 원격 제어 + AI Studio

> 오늘은 절반이 **컴퓨터 화면 작업**(구글)이에요. 보드는 후반에 등장!
> 최대 고비는 **"배포"** — 순서대로만 하면 한 번에 됩니다.

---

## 실습 1. 구글 시트 만들기

**👣 순서**
1. 구글 드라이브 → `새로 만들기 → Google 스프레드시트`
2. 왼쪽 위 제목을 `팀1_데이터`로 변경
3. **1행에 제목 입력**: A1부터 차례로 `시각` `온도` `습도` `조도` `소음`
4. 아래 탭 이름이 **`시트1`** 인지 확인 (코드와 철자가 같아야 해요!)

> 💡 이 1행 제목은 **절대 지우지 마세요** — 데이터가 그 아래로 쌓입니다.

## 실습 2. Apps Script에 첫 서버 코드 넣기

**📄 코드**: [`01_apps_script_basic.gs`](../02_sessions/04_cloud_expansion/code/01_apps_script_basic.gs)

**👣 순서**
1. 시트 메뉴 `확장 프로그램 → Apps Script` → 새 탭이 열려요
2. 원래 있는 `function myFunction(){}` **전부 지우기**
3. 코드 복사 → 붙여넣기 → **아무것도 수정하지 않아도 됩니다!**
   (`// 주석`은 설명 메모예요 — 채워 넣는 곳 아님. `e`는 요청이 올 때 구글이 자동으로 채워줘요)
4. Ctrl+S 저장 → 프로젝트 이름을 `팀1_서버`로

## 실습 3. 웹앱 배포 — ★오늘의 최대 고비★

**👣 순서**
1. 오른쪽 위 `[배포] → [새 배포]`
2. 톱니바퀴 → 유형 **'웹 앱'** 선택
3. "액세스 권한이 있는 사용자" → **★모든 사용자★** 로 변경 ← 실패 원인 1위! '나만'이면 보드가 거부당해요
4. `[배포]` → 권한 요청 창: 계정 선택 → `고급 → (안전하지 않은 페이지로) 이동 → 허용`
   (내가 만든 앱이라 안전해요 — 구글의 형식적 확인)
5. **'웹 앱 URL' 복사** → 팀 메모장에 저장! 끝이 **`/exec`** 인지 확인

## 실습 4. 브라우저로 서버 개통 테스트

**🎯 무엇을**: 보드 없이, 가짜 값을 손으로 보내서 서버가 잘 받는지 시험

**👣 순서**
1. 메모장에서 주소 조립: `복사한URL` 뒤에 이어 붙이기 →
   ```
   복사한URL?temp=25&humi=48&light=800&sound=120
   ```
   (`?` = 지금부터 데이터 / `temp=25` = temp라는 이름표에 25 / `&` = 구분자. 숫자는 아무거나 OK!)
2. 통째로 브라우저 주소창에 붙여넣기 → Enter

**✅ 성공이면**: 화면에 **OK** + 시트 탭에 `시각·25·48·800·120` **한 줄이 자동으로!** 숫자 바꿔 한 번 더 → 줄이 또 늘어나요

**🚨 안 되면**
- "권한이 없습니다" → 실습 3의 3번(모든 사용자) 다시 → `[배포 관리] → 연필 → 새 버전 → 배포`
- 시트에 빈 칸만 기록 → 주소의 `temp` 철자가 코드의 `e.parameter.temp`와 다름 (대소문자까지!)

## 실습 5. '설정' 탭 만들기 (원격 제어 준비)

**그림으로 보기**: [시트 탭 구성 안내](../images/sheet_tabs_guide.png)

**👣 순서**
1. 시트 **왼쪽 아래 ＋ 버튼** → 새 탭 생성
2. 새 탭 이름 더블클릭 → **`설정`** 으로 변경
3. 그 탭의 **A1 한 칸**에 `ON` 입력 (따옴표 없이, **대문자**)

> ⚠ 자주 하는 오해 3가지
> - **시트1 탭의 '시각'을 지우는 게 아니에요!** 명령은 **별도의 '설정' 탭**에
> - A1에는 `ON/OFF` 둘 다가 아니라 **한 번에 하나만** — 켤 땐 ON, 끌 땐 OFF로 바꿔 쓰기
> - `on`·`On`은 무반응 — 코드의 `if (cmd == "ON")`과 대소문자까지 같아야

## 실습 6. 서버 코드를 완성판으로 교체

**📄 코드**: [`03_apps_script_full.gs`](../02_sessions/04_cloud_expansion/code/03_apps_script_full.gs) (mode=cmd·set·latest 분기 포함)

**👣 순서**
1. Apps Script 편집기의 기존 코드 **전부 지우기** → 완성판 붙여넣기 → 저장
2. **재배포**: `[배포] → 배포 관리 → 연필(수정) → 버전: 새 버전 → 배포`
   ← 이걸 안 하면 코드를 바꿔도 **반영이 안 됩니다!** (오늘 제일 많이 잊는 것)
3. 테스트: 브라우저에 `URL?mode=cmd` → 화면에 **ON** (A1의 값)이 보이면 성공!

## 실습 7. ESP32를 클라우드에 연결 (Station 모드)

**🔌 배선**: [셀 3개 동시 연결 그림](../images/wiring_env_monitor.png) — DHT(Digital '6'칸=27) + 빛(A3=34) + 소리(A2=35)

**📄 코드**: [`04_esp32_cloud_template.ino`](../02_sessions/04_cloud_expansion/code/04_esp32_cloud_template/04_esp32_cloud_template.ino)

**👣 순서 — 바꿀 곳은 딱 3줄!**
1. `WIFI_SSID` = 교사 핫스팟 이름 (⚠ **2.4GHz만!** 5GHz는 ESP32가 못 붙어요)
2. `WIFI_PASS` = 핫스팟 비밀번호
3. `URL` = 실습 3에서 복사한 웹앱 URL (끝이 /exec)
4. 업로드 → 시리얼 모니터(115200) 열기 → `WiFi 연결 완료!` 확인

**✅ 성공이면**
- 시트1 탭에 **10초마다 한 줄씩 자동으로** 쌓여요 (박수!)
- 설정 탭 A1에 `ON` → **최대 10초 뒤** 보드 LED 켜짐 / `OFF`로 바꾸면 꺼짐
  (즉시 안 바뀌는 게 정상 — 10초마다 확인하는 '폴링'이라서)

**🚨 시트에 안 쌓이면**: ① URL 직접 열어 OK 나오나 ② 시리얼에 200 찍히나 ③ 코드에 `setFollowRedirects` 줄 있나 — 끊긴 구간만 고치면 돼요

## 실습 8. AI Studio로 웹앱 만들기 + 폰에 설치

**전체 가이드**: [AI Studio 웹앱 가이드](../01_docs/ai_studio_webapp_guide.md)

**👣 순서**
1. aistudio.google.com → **Build → New App**
2. [프롬프트 샘플](../02_sessions/04_cloud_expansion/code/05_ai_studio_prompt.md) 복사 → **`[우리 팀 웹앱 URL]`을 전부 실제 URL로 교체한 뒤** 붙여넣고 생성
   (⚠ 교체 없이 생성하면 AI가 자기 주소(…run.app)로 데이터를 받는 규칙을 지어내요 — 그 주소는 쓰지 않습니다!)
3. 앱이 만들어지면 반드시 확인: 카드에 시트의 최신 값이 나오는지 (안 나오면 채팅으로 "10초마다 [URL]?mode=latest 에서 값을 가져와 표시해줘"라고 URL 포함해 재지시)
4. 공유(Share) → 링크 생성 → 팀 채널로 전송
5. 각자 폰에서 링크 열기 → **홈 화면에 추가** (안드로이드: 메뉴⋮ / 아이폰: 공유⬆) → 앱 아이콘 완성!

**✅ 최종 성공 그림**: 폰 앱에서 온습도 확인 + LED 버튼으로 교실 불 제어 + 시트에 모든 기록 → **이게 팀 프로젝트의 뼈대 템플릿!**

> 💎 오늘의 3대 보물 백업: ① 웹앱 URL ② 시트 ③ ESP32 코드 — 팀 드라이브에!
