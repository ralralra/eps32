# Wemos D1 R32 처음부터 세팅하기 (완전 가이드)

> 이 문서 하나로 **빈 컴퓨터 → 첫 업로드 성공**까지 갑니다.
> 우리 수업의 모든 실습은 이 세팅이 끝난 상태에서 시작해요.

---

## 0. 우리 보드 소개 — Wemos D1 R32

| | 아두이노 우노 | **Wemos D1 R32 (우리 보드)** |
|---|---|---|
| 생김새 | 우노 모양 | **우노와 똑같은 모양** (실드 호환!) |
| 두뇌 칩 | ATmega328 | **ESP32** (훨씬 빠름) |
| WiFi | ❌ 없음 | ✅ **칩에 내장** |
| 블루투스 | ❌ 없음 | ✅ 내장 |
| 아날로그 해상도 | 0~1023 (10비트) | **0~4095 (12비트)** |
| USB 칩 | ATmega16U2 등 | **CH340** ← 드라이버 필요! |
| 동작 전압 | 5V | **3.3V** (핀에 5V 직결 금지) |

**핵심 한 줄**: 우노 모양이라 **고릴라 실드를 그대로 끼울 수 있고**, ESP32라서 **WiFi로 스마트폰과 통신**할 수 있는 보드예요.

⚠️ 주의 2가지
- 보드에 인쇄된 핀 이름(D2, A0 …)과 코드에서 쓰는 **GPIO 번호는 다릅니다** → [`pinmap_wemos_d1_r32.md`](pinmap_wemos_d1_r32.md) 참고
- ESP32는 **2.4GHz WiFi만** 됩니다 (5GHz 공유기/핫스팟 연결 불가)

## 1. CH340 드라이버 설치 (최초 1회)

Wemos D1 R32는 USB 통신에 **CH340** 칩을 씁니다.
드라이버가 없으면 **포트(COM)가 아예 안 보여요** — 세팅 실패 원인 1위!

1. 보드를 꽂고 드라이버를 설치하면 포트잡기 쉬워요 
2. 드라이버 파일: `CH341SER` 둘다 다운로드 및 설치 [`ch340`](esp32-class-projects/CH34x_Install_Windows_v3_4.zip) [`ch341`](esp32-class-projects/CH341SER.zip)
3. 설치 **(install)** - 실패시 uninstall 후 다시 install
4. **확인 방법 (Windows)**: 보드를 USB로 연결 → `장치 관리자` → `포트(COM & LPT)` →
   `USB-SERIAL CH340 (COM3)` 같은 항목이 보이면 성공 ✅
   - 노란 느낌표(⚠)가 붙어 있으면 드라이버 재설치
5. **Mac**: 최신 macOS는 대부분 기본 인식. 안 되면 CH34x 드라이버 설치 후 재부팅  

> 💡 USB 케이블은 반드시 **데이터 전송용**이어야 해요. 충전 전용 케이블은 전원 LED는 켜지지만 포트가 안 잡힙니다.

## 3. ESP32 보드 지원 설치 (최초 1회)

IDE는 처음엔 ESP32를 몰라요. "보드 지원 패키지"를 설치해야 인식합니다.

1. `파일` → `환경설정(Preferences)` 열기
2. **추가 보드 매니저 URL** 칸에 아래 주소 붙여넣기 (오타 방지: 배포자료에서 복사!)
   ```
   https://espressif.github.io/arduino-esp32/package_esp32_index.json
   ```
3. `툴` → `보드` → `보드 매니저` 열기
4. 검색창에 `esp32` 입력 → **esp32 by Espressif Systems** 설치
   - 다운로드에 몇 분 걸려요 — 기다리는 동안 키트 구경!
5. 설치 완료 후 `툴` → `보드` 목록에 **esp32** 항목이 생겼으면 성공 ✅

> 💡 수업 시간 절약을 위해 **교사 사전 설치를 권장**합니다. 학생 화면에서는 "설치돼 있는지 확인"만 해도 충분해요.

---

## 4. 보드·포트 선택

1. 보드를 USB로 연결
2. `툴` → `보드` → `esp32` → **ESP32 Dev Module** 선택
   - 우리 보드(D1 R32)는 이 설정으로 동작해요. 목록에 **WEMOS D1 R32** 가 있으면 그걸 선택해도 됩니다.
3. `툴` → `포트` → `COM○` 선택
   - **내 보드 찾는 요령**: 케이블을 뽑고 목록 확인 → 다시 꽂고 확인 → **새로 나타난 번호**가 내 보드!
4. 툴 메뉴 하단에 `ESP32 Dev Module on COM○` 이 표시되면 준비 완료

## 5. 첫 업로드 — 내장 LED 깜빡이기

실드 없이 **보드만으로** 개발환경을 검증합니다. D1 R32의 내장 LED는 **GPIO 2번**이에요.

```cpp
// 첫 업로드 테스트 — 내장 LED 깜빡이기
#define LED 2            // D1 R32 내장 LED

void setup() {
  pinMode(LED, OUTPUT);  // 2번 핀 = 출력
}

void loop() {
  digitalWrite(LED, HIGH); // 켜기
  delay(1000);             // 1초 대기
  digitalWrite(LED, LOW);  // 끄기
  delay(1000);
}
```

1. 위 코드를 새 스케치에 입력
2. **화살표(→) 버튼** 클릭 — 체크(✓)는 검사만, 화살표가 진짜 업로드!
3. 하단 검은 콘솔에서 `Connecting...` → `Writing...` → 퍼센트 진행 관찰
4. `업로드 완료` + **보드의 파란 LED가 1초 간격으로 깜빡이면 합격!** 🎉

> 💡 D1 R32는 대부분 **자동으로 업로드**됩니다.
> 드물게 `Connecting...`에서 오래 멈추면 그 글자가 보일 때 **BOOT(IO0) 버튼을 2~3초** 눌러 주세요.

## 6. 고릴라 실드 + 고릴라셀 결합

복잡한 배선 없이, 케이블을 포트에 꽂으면 끝!

1. **고릴라 실드** = 포트 어댑터. 보드 위에 끼우는 판으로, 센서 셀을 꽂는 포트가 줄지어 있어요
2. **고릴라셀** = 센서 모듈 16종
   (소리·빛·온습도·물온도·토양습도·초음파·진동·무게·대기압·CO2·미세먼지·홀·자이로·심박 + LCD)
3. **결합 시 주의**: USB 포트 방향을 맞추고, 핀이 휘지 않게 **수직으로 천천히** 끼워요
4. **연결 규칙 딱 하나만 기억**:

   > 🔑 **아날로그 센서 셀은 A2~A5 위치 포트에!** (GPIO 35·34·36·39)
   > WiFi를 켜면 A0·A1 자리(ADC2 핀)는 아날로그 읽기가 멈추기 때문이에요.
   > 자세한 이유와 전체 핀맵 → [`pinmap_wemos_d1_r32.md`](pinmap_wemos_d1_r32.md)

5. LCD 셀은 **I2C(SDA·SCL) 포트**에 연결 (GPIO21·22)
6. 수업 중에는 배포된 **포트-핀맵 카드**를 항상 옆에 둬요 — 코드의 핀 번호는 이 카드 기준!

## 7. 라이브러리 설치 (1회차 수업 전 권장)

LCD·온습도 같은 장치는 "라이브러리(미리 만든 도구상자)"가 있어야 움직여요.

1. `스케치` → `라이브러리 포함` → `라이브러리 관리` 열기
2. `LiquidCrystal I2C` 검색 → 설치 (LCD용)
3. `DHT sensor library` 검색 → 설치 (온습도용 — Adafruit, 의존성 함께 설치)
4. 인터넷이 느리면: `스케치` → `라이브러리 포함` → `.ZIP 라이브러리 추가` → 교사 USB의 zip 선택
5. **설치 확인**: `파일` → `예제` 목록에 새 항목이 보이면 성공

> 💡 `#include` 줄에서 에러가 나면 100% 라이브러리 미설치 — 이 단계로 돌아오면 돼요.

## 8. 자주 막히는 문제 해결소

| 증상 | 해결 |
|---|---|
| 포트(COM)가 목록에 없어요 | ① USB 케이블이 충전 전용인지 확인 → 데이터 케이블로 교체 ② **CH340 드라이버** 설치 (1번 항목) |
| `Connecting...`에서 멈춰요 | 그 글자가 보일 때 **BOOT(IO0) 버튼 2~3초** 누르기 |
| 업로드는 됐는데 동작 안 해요 | EN(리셋) 버튼 한 번 누르기, USB 재연결 |
| 보드가 뜨겁고 인식 안 돼요 | 즉시 뽑기! 실드 결합 방향·핀 밀림 확인 |
| WiFi 연결이 안 돼요 | 2.4GHz인지 확인 (ESP32는 5GHz 불가), 비밀번호 8자 이상 |
| 아날로그 값이 0/4095 고정 | 센서 셀이 **A2~A5 위치**인지, 코드 핀 번호가 핀맵 카드와 같은지 확인 |
| LCD에 아무것도 안 나와요 | ① 주소 `0x27` → `0x3F` 변경 ② 뒷면 파란 다이얼(대비) 조절 ③ I2C 스캐너 코드로 실제 주소 확인 |

> 🧭 태도 한 가지: **에러는 정보예요.** 막히면 메시지를 읽고, 위 표에서 증상을 찾으면 됩니다.

## 9. 실습 공통 규칙 5가지 (모든 회차 공통 — 꼭 기억!)

1. **코드는 통째로 복사** — IDE에서 `파일→새 스케치` → 전체 지우고 → 붙여넣기. 직접 타이핑하지 않아도 돼요
2. `//` 뒤의 글은 **설명 메모(주석)** — 뭘 채워 넣는 곳이 아니에요. 지워도, 그대로 둬도 됩니다
3. **아날로그 센서 셀은 A2~A5 위치** 포트에만 (A0·A1 자리는 WiFi를 켜면 아날로그를 못 읽어요)
4. 코드의 핀 번호는 실드에 인쇄된 숫자가 아니라 **변환된 GPIO** — [핀맵 그림](../images/gorilla_shield_x_d1r32_pinmap.png)의 노란 칸 숫자
5. 업로드 버튼은 **화살표(→)** — 체크(✓)는 검사만 하고 보드에 안 들어가요
