## ESP32 단계 (4~7)

### ⭐ `Brownout detector was triggered` 가 반복되고 부팅이 안 됨 (실전에서 제일 많이 막힘!)

시리얼에 아래가 계속 반복되고, 우리 코드 출력(`=== ESP32 출석 서버 ===`)은 한 번도 안 보이는 경우:
```
entry 0x400805f0
Brownout detector was triggered
rst:0xc (SW_CPU_RESET) ...
(다시 부팅 → 또 Brownout → 무한 반복)
```

업로드는 끝났는데 보드가 스케치를 정상 실행하지 못하고 리셋만 반복하는 상태입니다.

**✅ 검증된 해결 (이걸로 풀렸음)**: **IO0(BOOT) + RESET(EN) 버튼을 동시에 눌렀다 떼기**
1. 보드의 **IO0(BOOT)** 버튼과 **RESET(EN)** 버튼을 **둘 다 같이** 누른다
2. 두 버튼을 동시에 뗀다 (또는 RESET을 먼저 떼고 BOOT를 뗀다)
3. 보드가 깨끗하게 재부팅되며 스케치가 실행됨 → 폰 와이파이 목록에 **`ESP32-Attend`** 가 뜸
4. 뜨면 비번 `attend1234` 로 접속 → 브라우저 `http://192.168.4.1`

> 💡 Wemos D1 R32는 자동 리셋 회로가 가끔 매끄럽지 않아, 업로드 직후 스케치로 안 넘어가고
> 리셋 루프(Brownout 메시지 포함)에 빠질 수 있습니다. 이때 **버튼 두 개를 동시에 눌러 수동으로 재부팅**하면 정상 실행됩니다.

**그래도 안 되면 전원도 점검**:
- 노트북 USB 포트가 약하면 부팅 스파이크에서 진짜 Brownout이 날 수도 있음
- 휴대폰 충전기(5V 2A↑)/보조배터리 + 케이블로 켜보기, 노트북은 파란색(USB 3.0) 포트 사용
- 보드 5V↔GND 에 470µF~1000µF 콘덴서 추가하면 스파이크 흡수

> 🛡 step4~7 스케치에는 이미 안전장치로 `WRITE_PERI_REG(RTC_CNTL_BROWN_OUT_REG, 0);`
> (Brownout 감지기 끄기)가 `setup()` 첫 줄에 들어 있습니다. 정상 부팅 후 와이파이를 켤 때의
> 순간 전압 강하로 보드가 꺼지지 않게 막아줍니다. (단, 부팅 자체가 안 될 땐 위의 버튼 조작이 먼저입니다.)

### 폰이 ESP32 와이파이를 못 찾음
- 먼저 위 **Brownout/부팅 안 됨**이 아닌지 확인 (제일 흔함) → IO0+RESET 동시 누르기
- 시리얼 모니터에 `와이파이 이름: ESP32-Attend` 가 떴는지 확인
- 폰 와이파이 목록 새로고침
- 비밀번호 8자 이상인지 (`AP_PASS`)
- `AP_SSID` 를 **집 와이파이와 같은 이름으로 두지 말 것** — 폰이 진짜 공유기에 붙어 192.168.4.1 이 안 열림

### 와이파이는 붙었는데 페이지가 안 뜸
- 주소를 정확히: **http://192.168.4.1** (https 아님!)
- 폰이 "인터넷 없음" 이라며 다른 망으로 자동전환했을 수 있음 → 와이파이 설정에서 "계속 연결" 고정
- 일부 안드로이드는 자동으로 접속창(캡티브 포털)을 띄움 — 거기서 안 되면 크롬 주소창에 직접 192.168.4.1 입력

### 출석 버튼 눌러도 기록 안 됨 (5단계)
- 시리얼 모니터에 `출석: 홍길동` 이 찍히는지 확인 → 안 찍히면 `/check` 경로 등록(`server.on`) 확인
- 페이지 fetch 주소 오타 확인

### 전원 껐다 켜니 명단 사라짐 (6단계)
- `SPIFFS.begin(true)` 호출했는지
- 파티션에 SPIFFS 공간 있는지 (`Tools` → Partition Scheme: Default 4MB with spiffs)

### 여러 명이 동시에 접속하면 느림
- ESP32 AP는 보통 **4명** 정도 동시 접속 권장. 학급 전체가 한꺼번에 몰리면 순서대로
- 실제 수업에선 분단별로 나눠서 태그하거나, 시간차를 두면 안정적

## 업로드 단계

| 증상 | 해결 |
|---|---|
| `Connecting......____` 무한 | 보드 **BOOT 버튼** 누른 채로 Upload, `Writing` 시작되면 떼기 |
| 포트가 안 보임 | USB 드라이버(CP210x/CH340) 설치, 데이터 케이블인지 확인 |
| 업로드는 됐는데 시리얼 깨짐 | 모니터 baud 115200 확인 |

## 자주 하는 질문

**Q. 학생이 남의 이름으로 출석하면?**
A. 웹 출석의 한계입니다. 대리출석 방지가 중요하면 RFID 카드 방식(별도 단계)으로 확장하세요. 도입/수업 시연용으로는 웹 출석이 충분합니다.

**Q. 진짜 시간(날짜)은 어떻게 기록되나?**
A. 인터넷이 없어 ESP32는 시계가 없습니다. 그래서 **학생 폰의 시각**을 같이 보내서 기록합니다(코드에 `new Date()`). 폰 시계가 맞으면 정확합니다.

**Q. 반이 여러 개면?**
A. `AP_SSID` 를 `ESP32-1반`, `ESP32-2반` 처럼 보드마다 다르게 해서 여러 대 운영하면 됩니다.
