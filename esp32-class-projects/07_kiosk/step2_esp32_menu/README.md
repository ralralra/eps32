# 2단계 — ESP32가 키오스크 화면 보여주기

## 목표
폰/태블릿으로 ESP32에 접속하면 1단계 키오스크 화면이 뜬다. 상품 선택·장바구니·합계까지 동작.

## 핵심 아이디어
출석 4단계와 똑같아요 — 1단계 HTML을 ESP32 코드 안 문자열(`page`)에 넣고, 접속하면 보내줍니다.
지금은 결제(포인트)가 **브라우저 안(localStorage)** 에 있어요. 3단계에서 이걸 ESP32(기기)로 옮깁니다.

## 해보기
1. `step2_esp32_menu.ino` 업로드
2. 시리얼에 `키오스크: http://192.168.4.1` 확인
3. 폰/태블릿으로 `ESP32-Kiosk` 접속 (비번 `kiosk1234`) → http://192.168.4.1
4. 상품 담고 결제까지 되는지 확인

## 확인 ✅
- [ ] 폰/태블릿에 키오스크 화면이 뜬다
- [ ] 상품 담기·합계·결제가 동작한다
- [ ] (태블릿이면 진짜 키오스크처럼 보임)

## 잘 안 될 때
- 접속 문제 → 출석 트러블슈팅과 동일 (http, IO0+RESET)

## 한계 발견하기
지금은 포인트가 **각 기기 브라우저 안**에 따로 있어요. 그래서:
- 다른 폰으로 접속하면 포인트가 따로 놀고
- 판매 기록이 **어디에도 안 남아요**
→ 다음 단계에서 결제와 기록을 **ESP32(한곳)** 로 옮깁니다.

## 배운 것
- 웹서버로 키오스크 화면 제공 (출석에서 배운 것 재사용)
- 아직 결제/기록이 기기에 없음 → 공용 기기 저장의 필요성
