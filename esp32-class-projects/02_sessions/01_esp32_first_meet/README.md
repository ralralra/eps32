## 1교시 — 오리엔테이션

### IoT(사물인터넷)란?

**사물(Thing)** + **센서** + **인터넷 연결** = **스마트한 사물**

- 화분·매점 진열대·우산꽂이·교실 문 같은 평범한 물건이
- 주변을 '느끼고'(온도·밝기·무게·수분·문 열림)
- 느낀 것을 숫자 데이터로 바꿔 어디로든 보내면
- 스마트폰으로 화분의 목마름을 확인하는 세상이 돼요

> 💬 핵심 질문: 여러분 주변에서 "인터넷에 연결되면 더 편해질 물건"은?

### 우리가 만들 시스템의 전체 구조

```
📱 학생 스마트폰   ← 웹페이지에서 버튼 터치
      ▼
🌐 웹페이지 (HTML·CSS·JS)   ← 2~3회차에 직접 제작
      ▼
📡 ESP32 보드   ← 명령을 받아 장치를 움직임
      ▼
🔧 센서 셀 16종 · 7세그   ← 오늘 만나기 시작!
      ▼
📊 데이터 저장·통계 (CSV)   ← 4회차부터 연결
```

오늘은 맨 아래 장치부터 시작해서, **매주 한 층씩 올라가** 마지막엔 전체가 하나로 연결됩니다.

### 7주 로드맵

| 회차 | 주제 | 만드는 것 |
|:---:|---|---|
| **1** | ESP32 첫 만남 — LED·7세그·첫 센서 (오늘) | 카운트다운 타이머 |
| 2 | 센서 입력 + 웹 기본기(HTML·CSS·JS) | 접근 경보기 · 내 리모컨 웹페이지 |
| 3 | ESP32 웹서버 — 폰으로 보드 제어 | 내 손안의 리모컨 |
| 4 | 클라우드 확장 (구글 시트 저장) | 교실 환경 모니터 |
| 5 | 팀 프로젝트 기획 · 웹앱 제작 | 기획서 + 웹앱 1차 버전 |
| 6 | 핵심 기능 구현 + 통계 | 동작하는 우리 팀 시스템 |
| 7 | 테스트 · 발표 · 시연 | 최종 발표! |

### 팀 프로젝트 5가지 — 오늘 1·2지망 정하기 (확정은 4회차 후!)

| # | 프로젝트 | 한 줄 소개 |
|:---:|---|---|

| ② | 스마트 창업 키오스크 | 메뉴 주문 → 주문번호 7세그 표시 → 매출 통계 |
| ③ | 스마트 우산 대여 | 웹 QR로 대여·반납 → 초음파·무게로 거치 감지 → 이용 통계 |
| ④ | 스마트 화분 | 토양습도·빛·온습도 측정 → 급수 알림 → 기록·통계 |
| ⑤ | 스마트 출석·안전 | 웹 QR 체크인 → 홀센서로 문 열림 감지 → 출석부 통계 |

## 2교시 — 첫 코딩: 내장 LED

### ESP32는 무엇이 특별한가?

| 아두이노 우노 | ESP32 (우리 보드) |
|---|---|
| USB로 연결된 컴퓨터하고만 통신 | **WiFi가 칩 안에 내장!** |
| 인터넷 연결 불가 | 인터넷에 직접 연결 가능 |
| LED 깜빡이기 같은 단독 동작 | 스마트폰과 통신 가능 → IoT의 시작 |
| 코딩 입문용으로 훌륭 | 우노 모양이라 고릴라 실드도 그대로 결합 |

"WiFi 내장" 이 한 가지 차이가, 3회차에 **스마트폰으로 보드를 조종하는 마법**을 가능하게 해요.

### 따라하기 ① 보드를 컴퓨터에 연결하기

1. USB 케이블로 보드와 컴퓨터 연결 → 전원 LED 확인
2. Arduino IDE 실행
3. `툴` → `보드` → `esp32` → **ESP32 Dev Module** 선택 (우리 D1 R32는 이 설정으로 동작)
4. `툴` → `포트` → `COM○` 선택 — 케이블을 뽑았다 꽂으면 **새로 나타나는 번호**가 내 보드!
5. 업로드 테스트: 내장 LED 깜빡이기 (**2번 핀**) → [`code/01_blink/01_blink.ino`](code/01_blink/01_blink.ino)

> 🔧 포트가 안 보이면? CH340 드라이버 문제일 가능성 → 맨 아래 '문제 해결소' 참고

### 따라하기 ② 블록코딩으로 내장 LED 깜빡이기

블록은 문법 오류가 '아예 불가능'해요 — 오직 **순서와 논리**에만 집중!

1. 블록코딩 도구 실행, 보드 연결 상태 **초록색** 확인
2. `[시작하기]` 아래에 `[LED 켜기]` 블록 — 핀 번호는 **2** (D1 R32 내장 LED)
3. 실행 → 보드의 파란 LED가 켜져요! (첫 번째 성공 🎉)
4. `[1초 기다리기]` → `[LED 끄기]` 추가, 반복 블록으로 감싸 깜빡이게
5. **'C 코드 보기'** 클릭 → 내 블록이 C 코드로 어떻게 표현되는지 관찰

**블록 1개 = 코드 1줄.** 이 대응을 눈으로 확인하면 C 코드는 '낯선 언어'가 아니라 '블록의 다른 표기'가 돼요.

| 블록 | C 코드 |
|---|---|
| [시작하기] (1번만) | `void setup() { }` |
| [반복하기] (무한) | `void loop() { }` |
| [LED 켜기] | `digitalWrite(LED, HIGH);` |
| [1초 기다리기] | `delay(1000);` |

### 🎯 미션 ① 우리 팀 LED 신호 만들기

**목표**: 내장 LED의 깜빡임 패턴으로 팀만의 신호 3가지
(예: 대기=천천히 3번 · 작업중=빠르게 5번 · 완료=길게 1번)

완성 조건:
- [ ] 블록코딩으로 먼저 완성하고 동작 확인
- [ ] 'C 코드 보기'로 같은 코드 확인
- [ ] 그 코드를 Arduino IDE에 옮겨 직접 업로드해도 같은 동작
- [ ] 팀원 모두 한 번씩 업로드 성공 + 신호의 의미를 설명할 수 있음

> 💡 힌트: '빠르게'는 delay를 줄이면 된다(예: 150). 모스 부호처럼 짧음·긺의 조합이 곧 '정보' — 6회차 알림 설계의 씨앗! 예시: [`code/02_team_signal/02_team_signal.ino`](code/02_team_signal/02_team_signal.ino)

## 3교시 — 7세그먼트: 보드가 숫자로 말해요

### 최초 1회 준비 — 라이브러리 설치

1. `스케치` → `라이브러리 포함` → `라이브러리 관리`
2. `TM1637` 검색 → **TM1637Display (Avishay Orpaz)** 설치 (7세그먼트용)
3. `DHT sensor library` 검색 → 설치 (온습도용 — 2회차부터 사용, 미리!)
4. 인터넷이 느리면 `.ZIP 라이브러리 추가`로 교사 USB의 zip 선택
5. 확인: `파일` → `예제` 목록에 새 항목이 보이면 성공

> `#include` 줄에서 에러가 나면 100% 라이브러리 미설치 — 이 단계로 돌아오세요.
> "avr 아키텍처…" 경고가 떠도 **무시!** ESP32에서 정상 동작해요.

### 7세그먼트 — 4자리 숫자표시기

엘리베이터 층수, 전자시계의 그 숫자! 신호선 2가닥(DIO·CLK)으로 통신해요. **신호 2칸짜리 포트에 통째로** 연결합니다.

- 코드: [`code/03_7seg_hello/03_7seg_hello.ino`](code/03_7seg_hello/03_7seg_hello.ino)
- `setBrightness(0~7)`: 밝기 / `showNumberDec(숫자)`: 표시 (0~9999)
- 안 나오면: DIO·CLK 두 번호를 맞바꿔 재업로드 — **순서가 바뀌면 동작 안 해요**

### 갱신 — 계기판의 기본기

**변수·표시·갱신**, 이 셋이면 어떤 숫자든 실시간 표시할 수 있어요 → [`code/04_7seg_count/04_7seg_count.ino`](code/04_7seg_count/04_7seg_count.ino)

- `int count = 0`: 숫자를 담는 상자(변수)
- `showNumberDec(count)`: 상자 속 값을 표시
- `loop + delay`: 1초마다 갱신되는 화면
- 이 코드가 잠시 뒤 '카운트다운 타이머'의 심장이 돼요

### 🎯 미션 ② 팀 번호 보드

**목표**: 전원이 켜지면 7세그에 우리 팀 번호가 뜨고, 내장 LED가 팀 신호(미션①)로 인사

완성 조건:
- [ ] 7세그에 팀 번호 표시
- [ ] LED 팀 신호와 7세그 표시가 함께 동작
- [ ] `setup()`과 `loop()`의 역할 차이를 팀원에게 설명할 수 있음

> 💡 힌트: 7세그는 숫자 전용(0~9999)이에요. 팀 '이름'은 3회차에 웹페이지로 표시하게 됩니다!

## 4교시 — 첫 센서 + 종합 미션

### 첫 센서 — 빛센서 값을 7세그에 표시

읽고(센서), 판단하고(코드), 보여준다(7세그) — **IoT의 최소 단위** 완성!

- 코드: [`code/05_light_7seg/05_light_7seg.ino`](code/05_light_7seg/05_light_7seg.ino)
- 빛센서 셀은 **A2~A5 위치 포트**에! (실제 GPIO 번호는 포트-핀맵 카드 확인)
- `analogRead`: 밝기의 '정도'를 0(어두움)~4095(밝음) 숫자로
- 손으로 센서를 가려 봐요 — 숫자가 뚝 떨어져요!

> 🔑 왜 A2~A5 위치일까? **WiFi를 켜면 일부 핀(ADC2)의 아날로그 읽기가 멈추기 때문** — 3회차에 체감하게 돼요. → [`01_docs/pinmap_wemos_d1_r32.md`](../../01_docs/pinmap_wemos_d1_r32.md)

### 🎯 종합 미션 ③ 카운트다운 타이머

**목표**: 7세그에 10부터 0까지 1초씩 줄어드는 숫자 → 0이 되는 순간 표시가 깜빡이며 내장 LED 빠르게 5번 점멸

완성 조건:
- [ ] 7세그 카운트다운 (10 → 0)
- [ ] 0에서 깜빡임과 LED 점멸이 함께 동작
- [ ] 블록 또는 C 코드 중 팀이 선택 (C 도전 시 +성취)
- [ ] 심화: **빛센서를 가리면 타이머 일시정지!** (어두우면 count 유지)
- [ ] 완성 동작을 영상으로 촬영 → 오늘의 산출물

> 💡 힌트: `for(int i=10; i>=0; i--) { 표시; delay(1000); }` — 심화는 `if(analogRead(LIGHT) > 기준값)`일 때만 감소. 예시 답안: [`code/06_countdown_timer/06_countdown_timer.ino`](code/06_countdown_timer/06_countdown_timer.ino)

## 🔧 자주 막히는 문제 해결소 — 1회차

| 증상 | 해결 |
|---|---|
| 업로드 시 포트(COM)가 목록에 없어요 | USB 케이블이 '충전 전용'인지 확인 → 데이터 케이블로 교체. 그래도 안 되면 **CH340 드라이버** 설치 (교사 USB에 준비됨) |
| 업로드 중 `Connecting...`에서 멈춰요 | D1 R32는 보통 자동 진행 — 드물게 멈추면 그 글자가 보일 때 **BOOT(IO0) 버튼을 2~3초** 눌러요 |
| 7세그에 아무것도 안 나와요 | ① 케이블이 신호 2칸에 제대로 꽂혔는지 확인 ② 코드의 DIO·CLK 번호가 꽂은 칸의 GPIO와 맞는지(예: '6'칸=27·'7'칸=14) ③ 두 번호를 맞바꿔 재업로드 ④ TM1637 라이브러리 설치 확인 |
| 센서 값이 0 또는 이상한 값만 나와요 | 십중팔구 포트 위치·핀 번호 문제 — 포트-핀맵 카드와 코드의 `#define` 번호를 대조하고, 아날로그 셀은 **A2~A5 위치**인지 확인 |
