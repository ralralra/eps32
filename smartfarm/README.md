# 🌱 스마트팜 프로젝트 — 에듀이노 키트 × Wemos D1 R32 × Apps Script × AI Studio 앱

> 출석체크·교실 환경 모니터링에서 배운 IoT 구조를 그대로, 이번엔 **강낭콩을 키우는 스마트팜**으로!
> **에듀이노 스마트 팜 키트**의 센서·장치를 쓰되 보드는 **Wemos D1 R32 (ESP32)** — WiFi 내장이라
> 키트의 블루투스 모듈이 필요 없고, 앱은 앱인벤터 대신 **AI Studio(Build → New App)** 로 만들어 폰에 설치해요.
> 키트 원본 강의자료: [`강의자료_에듀이노 스마트 팜 키트.pdf`](강의자료_에듀이노%20스마트%20팜%20키트.pdf)

## 키트 강의자료 ↔ 우리 방식, 뭐가 달라지나

| 키트 강의자료 (아두이노 우노) | 우리 방식 (D1 R32) |
|---|---|
| 우노 + 센서 확장 쉴드 + 블루투스 모듈(D2·D3) | **D1 R32 + 센서 확장 쉴드** — 블루투스 모듈 없이 WiFi로! |
| 앱인벤터 앱 4종 (블루투스로 보드와 1:1 연결) | **AI Studio 앱 1개** — 인터넷만 되면 어디서든, 여러 폰에서 동시에 |
| 값 확인은 앱·시리얼·LCD | 값이 **구글 시트에 자동 기록** → 그래프·관찰일지까지 |
| 핀 번호 = 우노 기준 (A0, D9 …) | **GPIO로 변환해서 사용** (아래 변환표) |

앱인벤터 4종(토양습도 측정 · 온습도 측정 · LED 제어 · 쿨링팬 제어)의 기능은
AI Studio 앱 하나에 전부 들어갑니다 → [앱 프롬프트](01_docs/ai_studio_prompt.md)

## 시스템 전체 구조

```
🌱 스마트팜 키트 (강낭콩 화분)
   │  토양수분 · 온습도(DHT11)                  ← 입력(센서)
   ▼
📡 Wemos D1 R32 (ESP32) + 센서 확장 쉴드
   │  쿨링팬×2 · 네오픽셀(생장등/보온등)         ← 출력(장치)
   │  ① 10초마다 측정값 업로드   ② 명령 폴링 ("할 일 있어?")
   ▼                             ▲
🧾 Apps Script 웹앱 (/exec) ─────┘
   ▼
📊 구글 시트  ←  기록 탭(자동 쌓임) + 설정 탭(명령 A1 · 기준값 B1)
   ▲
📱 AI Studio 앱 (폰 홈 화면에 설치)
      토양습도·온도·습도 확인 · 팬/LED 제어 · 자동모드 · 물주기 알림
```

교실 환경 모니터링(4회차)과 **완전히 같은 구조**예요. 센서와 출력장치만 바뀌었을 뿐!

## 부품 구성 + 핀 변환표 (키트 강의자료 → D1 R32)

확장 쉴드는 우노 모양이라 D1 R32에 그대로 적층돼요. 단, **꽂는 줄과 코드의 번호**가 강의자료와 달라지는 곳이 있어요:

| 부품 | 키트 자료의 자리 | **우리가 꽂는 자리** | 코드의 번호 | 왜 바꾸나 |
|---|:---:|:---:|:---:|---|
| 토양수분 감지 모듈 | A1 | **A2 줄** | `35` | A1 자리(GPIO4)는 WiFi 켜면 못 읽어요 |
| 온·습도(DHT11) | A0 | **D2 줄** | `26` | A0 자리(GPIO2)는 부팅 문제 가능 |
| 쿨링팬 (모터드라이버) | D4·D5·D6·D7 | 그대로 | `17·16·27·14` | 자리는 같고 번호만 변환 |
| 네오픽셀 RGB LED | D9 | 그대로 | `13` | 번호만 변환 |
| LCD (I2C) | A4·A5 | ⚠ 사용 안 함 | — | D1 R32에선 그 자리가 입력 전용 — 화면 역할은 앱이! |
| 블루투스 모듈 | D2·D3 | ⚠ 사용 안 함 | — | WiFi 내장이라 불필요 |

> 전체 변환 원리 → [우노 라벨 ↔ GPIO 변환표](../esp32-class-projects/01_docs/pinmap_wemos_d1_r32.md)
> ⚠ 아날로그 값 범위도 달라요: 우노 0~1023 → **ESP32 0~4095**
> ⚠ 팬이 힘없이 돌면 키트의 **배터리 홀더**로 외부 전원을 공급하세요 (강의자료 방식 그대로)

## 자동 제어 규칙 (강낭콩 발아 기준 — 생육정보: 21.6~25.8℃ · 습도 50%)

| 조건 | 동작 | 근거 |
|---|---|---|
| 온도 > 26℃ | 쿨링팬 ON | 발아 적정온도 초과 |
| 온도 < 21℃ | 네오픽셀 따뜻한 빛 | 한파 피해 방지 (키트 자료 방식) |
| 습도 > 80% | 쿨링팬 ON | 곰팡이 방지 환기 |
| 토양습도 > 70% | 네오픽셀 따뜻한 빛 | 과습 말리기 (키트 자료 방식) |
| 토양습도 < 기준(앱에서 설정) | **앱·시리얼에 "물 주세요!"** | 키트엔 펌프가 없어요 — 사람이 급수! |

## 있으면 더 좋은 부품 추천 (키트에 없는 것)

| 부품 | 왜 좋은가 | 난이도 |
|---|---|:---:|
| **미니 워터펌프 + 릴레이 모듈** | "물 주세요 알림"을 **자동 급수**로 업그레이드 — 스마트팜의 완성 | ★★ |
| **수위 센서** | 펌프를 달았다면 필수! 물탱크 마름 감지 (펌프 공회전 보호) | ★ |
| **조도(CDS) 센서** | 햇빛 양 기록 → "빛과 성장" 관찰일지가 풍부해져요 | ★ |
| **서보 모터 (SG90)** | 환기창 개폐 데모 — 팬과 다른 방식의 온도 제어 비교 | ★★ |
| **부저** | 물 부족·고온 경보음 (앱 알림과 이중화) | ★ |

> pH·EC(양액 농도) 센서는 비싸고 보정이 어려워 수업용으론 비추천이에요.
