# 3단계 — 재고 차감 + 판매 기록

## 목표
팔 때마다 **재고를 1개 줄이고**, 판매 내역을 보드에 **저장**한다. 품절이면 못 팔게 막고, 재고 부족이면 알림.

## 원리
- 상품마다 재고 숫자를 가지고 있다가, 팔리면 `productStock[i]--`
- 판매 1건을 `/sales.csv` 파일에 한 줄씩 추가 (출석 6단계의 저장과 같은 방식!)

```cpp
productStock[i]--;                  // 재고 차감
logSale(productName[i], price);     // 파일에 기록
if (productStock[i] <= 0) ...       // 품절 처리
```

## 해보기
1. 상품표 UID를 내 카드로 수정 (2단계와 동일)
2. 시작 재고(`productStock`)를 원하는 수로
3. 보드 설정: Partition Scheme = **Default 4MB with spiffs**
4. 업로드 → 시리얼 모니터
5. 같은 상품을 재고보다 많이 찍어보기 → "품절!" 막히는지
6. 재고가 2개 이하로 떨어지면 "재고 부족" 알림 뜨는지

## 확인 ✅
- [ ] 팔면 "남은재고 N" 이 줄어든다
- [ ] 재고 0이면 "품절! 판매 불가"
- [ ] 재고 ≤2 면 "재고 부족" 경고
- [ ] (4단계에서 확인) 전원 꺼도 판매기록이 남는다

## 배운 것
- 숫자(재고)를 상태로 관리하고 조건으로 막는다
- 판매를 파일에 기록 = 나중에 통계의 재료
- 출석 프로젝트의 "저장" 개념이 매점에도 똑같이 쓰임
