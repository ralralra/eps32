# ESP32 스마트 프로젝트 수업 — 기획 Spec

> 고등학교 프로젝트 수업 / ESP32 프로토타입 제작
> 본 문서는 Claude Code에서 코드 구현을 시작하기 위한 **기획 단계 명세서**이다.
> 3D 모델링(케이스·거치대 등)은 별도 진행 예정. 프로토타입이므로 테스트용 이미지 또는 대표 학생 얼굴 사용 가능.

---

## 0. 공통 전제 및 기술 스택

**하드웨어 기본 구성**
- 메인보드: ESP32 DevKit (WiFi / Bluetooth 내장)
- 통신: WiFi → 서버(또는 로컬 PC)로 데이터 전송
- 개발환경: Arduino IDE 또는 PlatformIO (C/C++), 학생 수준에 맞게 Arduino 권장

**서버 / 데이터 (택1, 수업 난이도에 맞게)**
- 입문: ESP32 내장 메모리(SPIFFS) 또는 Google Sheets 연동(HTTP)
- 표준: 간단한 웹서버(Node.js / Python Flask) + SQLite/CSV
- 데이터는 모두 "이벤트 로그(누가·무엇을·언제)" 형태로 통일하면 통계가 자동 산출됨

**프로토타입 한계선 (3대 원칙)**
1. **실제 금전 거래 금지** → 모든 결제는 가상 포인트/모의결제로 대체
2. **정밀 위치추적 금지** → GPS·실내측위 대신 고정형 리더(특정 지점 통과 감지)로 대체
3. **다중 RFID 동시인식 금지** → 태그는 1개씩 단건 스캔 (UHF RFID는 고가·부적합)

## 1. 스마트 매점

**목표:** RFID 기반 셀프 계산 + 재고/판매 데이터 자동 관리

**기능 범위**

| 기능 | 판정 | 구현 방식 |
|------|:---:|------|
| RFID 상품 자동계산 | 부분가능 | RC522로 상품 태그 **1개씩 단건 스캔**, 가격 합산 |
| 재고 자동관리 | 가능 | 판매 시 수량 차감, 임계치 이하 시 알림 |
| 인기상품 분석 | 가능 | 판매 로그 집계 → 상위 품목 산출 |
| 폐기예상 알림 | 가능 | 상품별 유통기한 등록 → D-day 카운트 알림 |

**필요 부품:** RC522 RFID 리더, RFID 태그/카드(상품 수만큼), ESP32, LCD(I2C 1602/2004) 또는 OLED, 부저(선택)

**데이터 구조 (예시)**
- 상품: `{상품ID, 이름, 가격, 재고수량, 유통기한}`
- 판매로그: `{판매ID, 상품ID, 수량, 판매시각}`

**제외/주의:** 장바구니 통째 동시 인식(❌) → "한 개씩 찍는 셀프 계산대"로 한정

---

## 2. 스마트 창업 키오스크

**목표:** 매점 과자 판매를 위한 무인 키오스크 (창업 교육 연계)

**기능 범위**

| 기능 | 판정 | 구현 방식 |
|------|:---:|------|
| 과자 판매 UI | 가능 | 터치스크린 또는 버튼 + 화면으로 상품 선택 |
| 무인 결제 | 모의결제로 대체 | 가상 포인트 카드(RFID) 잔액 차감 방식 |
| 재고 관리 | 가능 | 판매 시 차감 (매점 과제와 동일 로직 재활용) |
| 매출 분석 | 가능 | 판매 로그 집계 → 일/품목별 매출 |

**필요 부품:** ESP32, 터치 TFT 디스플레이(또는 일반 TFT + 버튼), RC522(포인트카드용), (선택) 서보로 상품 배출 데모

**제외/주의:** 실제 카드사·PG 연동, 현금 처리(❌) → **모의결제 전용**. 교육 포인트는 "결제 흐름의 이해"

> 매점(1번)과 키오스크(2번)는 데이터 구조가 거의 동일 → 한 팀이 묶어서 진행하면 효율적

## 3. 스마트 우산 대여 ⭐ (단순화 확정안)

**목표:** 위치확인은 제외. **누가 빌리고 누가 반납했는지**만 기록하는 대출/반납 시스템 (도서관 대출 구조와 동일)

**기능 범위 (확정)**

| 기능 | 판정 | 구현 방식 |
|------|:---:|------|
| 사용자 식별 | 가능 | NFC 학생증 태그(PN532) 또는 RFID 학생카드 |
| 우산 식별 | 가능 | 우산 손잡이에 RFID 태그 부착 후 함께 스캔 |
| 대여 기록 | 가능 | "학생 + 우산 + 대여시각" 저장, 상태='대여중' |
| 반납 기록 | 가능 | 같은 방식 재태그 → 반납시각 기록, 상태='반납완료' |
| 이용 통계 | 가능 | 로그 자동 집계: 미반납 현황, 빈번 이용자 |

**동작 흐름**
1. 학생이 NFC 학생증 태그 → 사용자 인증
2. 우산 RFID 태그 스캔 → 어느 우산인지 식별
3. 서버에 대여 이벤트 기록
4. 반납 시 동일 절차 → 상태 변경

**데이터 구조**
- 대여로그: `{기록ID, 학생ID, 우산번호, 대여시각, 반납시각(null이면 대여중), 상태}`

**기록 예시**

| 우산번호 | 대여자 | 대여시각 | 반납시각 | 상태 |
|:---:|:---:|:---:|:---:|:---:|
| 3 | 김○○ | 6/28 14:20 | 6/28 17:05 | 반납완료 |
| 7 | 이○○ | 6/28 15:10 | — | 대여중 |

**필요 부품:** ESP32, PN532(NFC) 또는 RC522(RFID) 리더, 우산용 RFID 태그, LCD/OLED, (선택) 서보모터로 거치대 잠금 데모

**제외 확정:** 위치확인/GPS(❌)

> 식별 방식 결정 필요: **RFID 태그 통일**(학생증과 일관성 ↑) vs **번호 선택/QR**(부품 저렴). → RFID 통일 권장

## 4. 스마트 화분 관리 ⭐ (입문 최적, 강력 추천)

**목표:** 토양 수분 측정 + 자동 급수. 배선·코드가 단순하고 결과가 즉시 보여 성공경험용으로 최적

**기능 범위**

| 기능 | 판정 | 구현 방식 |
|------|:---:|------|
| 토양 수분 측정 | 가능(쉬움) | 수분센서 → ESP32 ADC로 값 읽기 |
| 자동 급수 | 가능(쉬움) | 임계치 이하 시 릴레이 ON → 워터펌프 동작 |
| (확장) 데이터 기록 | 가능 | 수분값 주기적 로그 → 그래프 |

**필요 부품:** ESP32, 토양수분센서(정전용량식 권장, 부식 적음), 릴레이 모듈, 소형 워터펌프 + 튜브, 5V 전원, (선택) DHT11 온습도

**데이터 구조:** `{측정시각, 수분값, 펌프동작여부}`

**제어 로직(의사코드)**
```
if 수분값 < 설정임계치:
    펌프 ON (n초)
    로그 기록
else:
    대기
```

## 5. 스마트 출석 및 안전관리

**목표:** NFC 학생증 기반 출입/출석 + 보호자 알림

**기능 범위**

| 기능 | 판정 | 구현 방식 |
|------|:---:|------|
| NFC 학생증 출입 | 가능 | PN532로 학생증 태그 인식 |
| 실시간 출석 | 가능 | 태그 시각 기록 → 서버로 전송 |
| 위험지역 접근 알림 | 재정의 필요 | 위험구역 출입구에 **고정형 리더** 설치 → 통과 감지 |
| 보호자 알림 | 가능 | WiFi → 이메일/메신저 webhook 전송 |

**필요 부품:** ESP32, PN532(NFC) 리더, 학생증(NFC 카드), LCD/OLED, (위험지역용) 추가 고정형 리더

**데이터 구조:** `{학생ID, 위치(리더ID), 이벤트(출입/통과), 시각}`

**제외/주의:** 학생 실시간 위치추적(❌) → "특정 지점 통과 감지"로 한정. 보호자 알림은 SMS 대신 이메일/메신저 webhook 권장(무료·간단)

> 3번 우산·5번 출석은 **NFC/RFID 태그 인식**이라는 동일 기술 기반 → 코드 모듈 재사용 가능

- `wifi_sender`: 서버 전송 (전 과제 공용)
